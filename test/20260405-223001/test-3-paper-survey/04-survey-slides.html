Streaming 3D Reconstruction with VGGT-based KV Cache Management

Paper Survey Report, 10 papers, 2026-04-05

This survey maps the landscape of streaming 3D reconstruction methods that extend visual geometry transformers (VGGT) with KV cache management for infinite-horizon operation. The field has rapidly evolved from pairwise pointmap regression (DUSt3R, 2024) through global geometry transformers (VGGT, 2025) to streaming systems that process arbitrarily long video sequences (InfiniteVGGT, 2026), with parallel developments in causal architectures and test-time adaptation.

Cluster Analysis

Cluster A: Pairwise Pointmap Regression

DUSt3R, MASt3R

These methods established that dense 3D geometry can be regressed directly from image pairs without explicit feature matching or epipolar geometry. MASt3R adds a feature-matching head that improves correspondence quality. The core consensus is that end-to-end regression outperforms classical two-stage pipelines for pairwise reconstruction. The limitation that drove subsequent work: pairwise methods require costly global alignment when scaling to many views.

Cluster B: Global Geometry Transformers

VGGT

VGGT resolved the multi-view scalability problem by processing all views jointly through a geometry-aware transformer. It produces globally consistent pointmaps, cameras, and depth in a single forward pass. This eliminated the need for post-hoc alignment but introduced quadratic memory cost in the number of views.

Cluster C: Streaming and KV Cache Methods

InfiniteVGGT, StreamVGGT

These methods address VGGT's quadratic memory bottleneck for long video sequences. InfiniteVGGT introduces geometry-aware KV cache pruning that retains tokens based on spatial coverage and confidence, enabling unbounded sequence processing with bounded memory. StreamVGGT focuses on latency optimization for real-time operation. The key shared insight: not all past tokens contribute equally, and intelligent pruning preserves reconstruction quality.

Cluster D: Causal and Recurrent Approaches

CUT3R, Point3R, TTT3R

These methods take an alternative path by processing frames causally (left-to-right), maintaining a recurrent state rather than attending to all past frames. CUT3R uses autoregressive prediction, Point3R accumulates point cloud state, and TTT3R adapts weights on-the-fly. The consensus is that causal processing naturally supports streaming, but the debate remains on whether it can match the global consistency of attention-based approaches.

Cluster E: Classical SLAM Baselines

ORB-SLAM, DROID-SLAM

ORB-SLAM demonstrated that keyframe management and loop closure enable long-horizon operation with bounded resources, an insight directly relevant to KV cache pruning. DROID-SLAM bridged classical and learned approaches by replacing hand-crafted features with learned dense features while retaining iterative optimization. These methods provide the operational baselines that streaming transformers aim to surpass.

Development Narrative

The trajectory from classical SLAM to streaming geometry transformers follows a clear pattern of removing hand-crafted components while progressively expanding the operational horizon. ORB-SLAM (2015) established the template for long-horizon 3D reconstruction through keyframe selection, map management, and loop closure, but relied entirely on hand-crafted ORB features and geometric solvers. DROID-SLAM (2021) replaced the feature extraction and matching stages with learned dense correspondences and differentiable bundle adjustment, achieving higher accuracy on standard benchmarks while preserving the iterative optimization paradigm.

DUSt3R (CVPR 2024) made a more radical departure by eliminating explicit geometric reasoning entirely: a transformer directly regresses 3D pointmaps from image pairs. MASt3R (ECCV 2024) refined this by reintroducing feature matching as an auxiliary output, improving correspondence quality without reverting to classical pipelines. However, both methods operate on pairs and require expensive global alignment to handle more than two views.

VGGT (CVPR 2025) resolved the multi-view problem by processing all images jointly through a geometry-aware transformer, producing globally consistent 3D in a single forward pass. This eliminated alignment costs but introduced quadratic memory scaling, limiting practical use to sequences of roughly 20-30 frames. The need to process long video sequences drove two parallel research directions: causal architectures (CUT3R, Point3R, TTT3R) that process frames sequentially with bounded state, and KV cache management approaches (InfiniteVGGT, StreamVGGT) that retain VGGT's global attention while pruning cached tokens to bound memory. InfiniteVGGT (2026) represents the current state-of-the-art by combining geometry-aware token pruning with confidence-based cache management, enabling thousands of frames with reconstruction quality approaching the full-attention baseline.

Open Questions

  • Cache pruning vs. causal state: No direct comparison exists between KV cache pruning (InfiniteVGGT) and causal recurrent state (CUT3R) under identical settings; which approach better preserves global consistency over thousands of frames remains empirically unresolved.
  • Loop closure in transformers: Classical SLAM systems handle revisited locations through explicit loop closure. Streaming transformers lack this mechanism, potentially accumulating drift in long trajectories with revisits.
  • Dynamic scenes: All surveyed methods assume static scenes. Extending KV cache management to handle moving objects and scene changes is unexplored.
  • Compute-quality tradeoff characterization: The relationship between cache budget (number of retained tokens) and reconstruction quality has been studied only for InfiniteVGGT's specific pruning strategy; the Pareto frontier across strategies is unknown.
