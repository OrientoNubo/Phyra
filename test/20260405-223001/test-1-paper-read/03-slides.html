3D Reconstruction Streaming KV Cache Transformer Visual Geometry

Problem & Motivation

Core tension: Offline models (VGGT) deliver high-quality 3D geometry but cannot operate on live streams. Streaming models (StreamVGGT) operate causally but their KV cache grows as $O(T \cdot P)$, exhausting GPU memory on long sequences.

The FlashAttention paradox: Optimized kernels avoid materializing the full $O(N^2)$ attention matrix for efficiency, but traditional pruning requires those attention weights to assess token importance. The tool needed to manage cache size prevents intelligent cache compression.

Gap: No existing method supports truly infinite-horizon 3D geometry understanding with bounded memory and stable quality.

Method: Diversity-Aware Rolling Memory

InfiniteVGGT resolves the paradox by replacing attention-based importance scoring with cosine similarity in key space, fully compatible with FlashAttention. Three integrated mechanisms:

  • Immutable Anchor Token: The first frame's KV cache is preserved permanently as a global coordinate reference, preventing coordinate drift.
  • Diversity-Quantified Token Retention: Redundant tokens are identified via negative cosine similarity to the mean key vector, without materializing attention matrices.
  • Layer-wise Adaptive Budget: Non-uniform storage budgets across transformer layers based on measured information diversity.

Key Formulas

Diversity score (per layer $l$, head $h$):

$$s_{\text{div}}^{(l,h)}(\hat{k}_i) = -\text{CosSim}(\mu^{(l,h)},\; \hat{k}_i)$$

where $\mu^{(l,h)} = \mathbb{E}[\hat{k}]$ over L2-normalized candidate keys. Tokens with the highest diversity scores (most dissimilar from the mean) are retained.

Budget allocation across layers:

$$p_{\text{bud}}^l = \frac{\exp(s_{\text{div}}^l / \tau)}{\sum_j \exp(s_{\text{div}}^j / \tau)}, \quad B^l = p_{\text{bud}}^l \cdot B_{\text{total}}$$

Default: $B_{\text{total}} = 25{,}000$ tokens per head, $\tau = 1.0$.

Phyra Critical Assessment

Strengths

  • Principled resolution of a real paradox: The FlashAttention-pruning conflict is a genuine engineering bottleneck, and the cosine-similarity proxy is both well-motivated and empirically validated.
  • Training-free: No fine-tuning required; the method modifies inference alone, lowering the barrier to adoption.
  • New benchmark contribution: Long3D (up to 9,545 frames) fills a real gap in long-sequence evaluation.

Weaknesses & Red Flags

  • Completion metric underperformance: The authors acknowledge this in the conclusion but do not provide a mechanistic explanation for why diversity-aware pruning degrades completion while improving accuracy.
  • First-frame dependency: The immutable anchor assumption is fragile; if the first frame is low-quality or unrepresentative, the entire coordinate system suffers. No robustness analysis is provided.
  • Limited domain evaluation: All benchmarks are indoor scenes. The claim of "infinite-horizon" capability lacks outdoor or mixed-domain validation.
  • Budget saturation analysis: Table 5 shows saturation at B=25,000, but no theoretical justification is given for why this value generalizes across scene types.

Key Takeaways

  • The core insight is that key-space cosine similarity serves as a viable, FlashAttention-compatible proxy for attention-based importance scoring.
  • The three-component design (anchor + diversity pruning + adaptive budgeting) addresses complementary failure modes: coordinate drift, information redundancy, and layer heterogeneity.
  • The method's training-free nature and constant per-frame cost make it immediately deployable on top of existing StreamVGGT checkpoints.
  • Open questions remain on first-frame robustness, outdoor generalization, and the completion metric gap.
