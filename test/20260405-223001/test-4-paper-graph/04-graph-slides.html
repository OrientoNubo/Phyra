Paper Analysis

DUSt3R (CVPR 2024)

Introduced pointmap regression as a unified representation for 3D reconstruction from image pairs. By predicting dense 3D pointmaps directly from a ViT encoder-decoder, DUSt3R eliminated the need for explicit feature matching, camera estimation, and triangulation as separate stages. This formulation enabled end-to-end training on large-scale data and established the foundation for subsequent geometry-aware transformers.

Wang et al. | Cluster: Pointmap Regression

MASt3R (ECCV 2024)

Extended DUSt3R by grounding local feature matching in 3D geometry. Rather than matching features in 2D image space, MASt3R predicted 3D-aware feature descriptors alongside pointmaps, enabling robust matching that respects geometric consistency. This parallel approach addressed the limitation that DUSt3R lacked explicit correspondences for downstream tasks like localization.

Leroy et al. | Cluster: Feature Matching

VGGT (CVPR 2025)

Proposed a feed-forward transformer that predicts cameras, point maps, depth, and 3D point tracks from image collections in a single forward pass. Unlike DUSt3R's pairwise formulation, VGGT processes multiple images jointly via cross-attention, achieving global consistency without iterative optimization. However, VGGT operates in batch mode and requires all frames to be available.

Wang et al. | Clusters: Geometry Transformer, Pointmap Regression

CUT3R (2025)

Adapted the VGGT-style architecture for causal, unposed temporal 3D reconstruction. CUT3R processes frames sequentially in an autoregressive manner, maintaining a fixed-size context of past observations to predict geometry for each new frame without requiring future frames. This enabled real-time applications but imposed a hard context window limit.

Wang et al. | Clusters: Geometry Transformer, Causal/Streaming

InfiniteVGGT (arXiv 2026)

Extended VGGT to handle unbounded video streams by introducing rolling memory with KV cache pruning. The architecture maintains a compressed memory of arbitrarily long histories while processing each new frame causally, removing CUT3R's fixed context limitation. InfiniteVGGT achieves superior accuracy and scalability by leveraging geometric importance scoring for memory management.

Yuan et al. | Clusters: Geometry Transformer, Causal/Streaming

Development Narrative

The trajectory begins with DUSt3R (CVPR 2024), which demonstrated that pointmap regression could replace the classical multi-stage 3D reconstruction pipeline. This insight spawned two parallel lines of work. MASt3R (ECCV 2024) addressed the lack of explicit correspondences by embedding feature matching into the 3D-aware framework, while VGGT (CVPR 2025) scaled the approach to multi-image collections through a unified feed-forward transformer.

VGGT's batch processing assumption became the next bottleneck. CUT3R (2025) responded by introducing causal autoregressive processing, enabling frame-by-frame reconstruction without future-frame dependency. However, CUT3R's fixed-size context window limited its applicability to long sequences. InfiniteVGGT (2026) resolved this by introducing rolling memory with geometric importance-based KV cache pruning, enabling reconstruction over unbounded streams while retaining information from the full sequence history.
