Cluster Analysis

Cluster A: Pointmap Regression

DUSt3R, MASt3R

The foundational approach: train a network to directly regress 3D pointmaps from image pairs. DUSt3R introduced the idea of bypassing explicit camera estimation by predicting dense 3D point coordinates in a shared reference frame. MASt3R extended this with learned local feature descriptors, enabling robust correspondence matching alongside geometry prediction.

Cluster B: Multi-View Geometry Transformer

VGGT (also bridges to Cluster C)

VGGT moved beyond the pairwise paradigm by processing an arbitrary number of input views through a single feed-forward transformer. It jointly predicts camera parameters, dense depth maps, and 3D pointmaps. This batch processing enables global geometric consistency that pairwise methods achieve only through expensive post-hoc optimization.

Cluster C: Streaming / Causal Processing

CUT3R, InfiniteVGGT (also bridges from Cluster B)

Both methods target real-time, frame-by-frame 3D reconstruction without requiring all frames upfront. CUT3R uses causal (autoregressive) attention masks so each frame can only attend to previously seen frames. InfiniteVGGT takes a different strategy: it maintains a rolling KV-cache that compresses past observations into a fixed-size memory, enabling processing of arbitrarily long sequences with bounded compute and memory.

Cross-Cluster Dynamics

The research lineage flows from pairwise pointmap regression (Cluster A) toward multi-view batch inference (Cluster B), then diverges into streaming architectures (Cluster C). VGGT and InfiniteVGGT are the bridge nodes: VGGT connects the pointmap foundation to the transformer-based multi-view approach, while InfiniteVGGT connects the batch transformer to the streaming paradigm.

Development Timeline

CVPR 2024

DUSt3R demonstrated that a single neural network can regress 3D pointmaps directly from stereo image pairs, removing the need for explicit feature matching or camera estimation pipelines. This reframed 3D reconstruction as a dense regression task.

ECCV 2024

MASt3R observed that DUSt3R's pointmaps lacked fine-grained correspondence. By jointly training a local feature head alongside the pointmap decoder, MASt3R enabled both dense geometry and reliable pixel-level matching within a single forward pass.

CVPR 2025

VGGT scaled the concept from pairs to batches. A feed-forward vision transformer ingests multiple views simultaneously and produces cameras, depth, and pointmaps in one shot. The key insight: global attention across views captures multi-view geometric constraints more efficiently than iterative pairwise alignment followed by bundle adjustment.

2025

CUT3R introduced causal attention masking to the geometry transformer, enabling autoregressive 3D reconstruction. Each new frame is processed using only previously observed context, making the approach suitable for online (streaming) applications where future frames are unavailable.

2026

InfiniteVGGT addressed the memory bottleneck of streaming reconstruction. By maintaining a rolling KV-cache that compresses historical geometric context into a fixed-size representation, it enables constant-memory, constant-time processing per frame, regardless of total sequence length. This approach supersedes CUT3R's fixed causal window for long-duration streaming scenarios.
