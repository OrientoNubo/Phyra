3D Reconstruction KV Cache Streaming Training-Free

Problem and Motivation

3D visual geometry understanding faces a fundamental trade-off between two paradigms:

  • Offline batch methods (e.g., VGGT, CVPR 2025 Best Paper): high-quality geometry prediction from fixed-length inputs, but unbounded memory and no support for live operation.
  • Online streaming methods (e.g., StreamVGGT): causal frame-by-frame processing, but the KV cache grows as $O(T \times P)$, eventually exhausting GPU memory.

The core paradox: FlashAttention avoids materializing full attention matrices for efficiency, yet traditional pruning methods require those attention weights to determine token importance. The tool needed to manage cache size prevents intelligent shrinking.

Key Insight

The authors observe that query tokens from the current frame assign near-identical attention weights to historically similar cached keys. This means redundant tokens are identifiable through key-space similarity alone, without materializing the $O(N^2)$ attention matrix.

Cosine similarity in key space serves as an attention-independent proxy for importance, fully compatible with FlashAttention. This resolves the computational paradox: cache management no longer conflicts with the inference kernel.

The proposed method is entirely training-free, requiring no fine-tuning of the base StreamVGGT model.

Method Overview: Three Integrated Mechanisms

  1. Immutable Anchor Token: The complete KV cache from the first frame is permanently preserved. VGGT establishes Frame 1 as the canonical coordinate system; losing it causes coordinate drift.
  2. Diversity-Aware Token Retention: For each layer and attention head, compute cosine similarity between each candidate key and the mean key vector. Tokens most dissimilar from the mean carry the most unique information and are retained.
  3. Layer-wise Adaptive Budget Allocation: Different transformer layers exhibit different information diversity patterns. Shallow layers show high diversity; early and deep layers show less. Budgets are distributed non-uniformly via softmax weighting.

Total budget: 25,000 tokens per head (fixed, regardless of sequence length).

Diversity Scoring

For each layer $l$ and attention head $h$:

Step 1: Compute the mean key vector over L2-normalized candidate keys:

$$\mu^{(l,h)} = \mathbb{E}_{\hat{k} \in \hat{K}_{t,\text{cand}}^{(l,h)}}[\hat{k}]$$

Step 2: Calculate the diversity score as the negative cosine similarity:

$$s_{\text{div}}^{(l,h)}(\hat{k}_i) = -\text{CosSim}(\mu^{(l,h)}, \hat{k}_i)$$

Step 3: Retain tokens with the highest diversity scores via TopK selection.

Tokens that are most dissimilar to the population mean carry the most unique geometric information. This formulation avoids any dependency on attention weights.

Layer-wise Adaptive Budget Allocation

Each layer's aggregate diversity $s_{\text{div}}^l$ is converted to a budget proportion via softmax:

$$p_{\text{bud}}^l = \frac{\exp(s_{\text{div}}^l / \tau)}{\sum_j \exp(s_{\text{div}}^j / \tau)}$$

where $\tau = 1.0$ is the temperature hyperparameter. The per-layer budget becomes:

$$B^l = p_{\text{bud}}^l \cdot B_{\text{total}}$$

This assigns more cache capacity to layers with higher information diversity (typically shallow layers that capture subtle inter-frame differences), and less to layers with redundant representations.

Critical Assessment

Strengths

  • Training-free integration: drops into StreamVGGT without fine-tuning, reducing adoption cost to zero.
  • Principled design: each component (anchor, diversity scoring, adaptive budget) has clear justification and is validated through ablation.
  • Practical scalability: bounded memory at 14.49 GB peak enables deployment on consumer GPUs for arbitrarily long sequences.
  • New benchmark contribution: Long3D fills a genuine evaluation gap for sequences beyond 2,000 frames.

Weaknesses

  • Completion metric underperformance: the authors acknowledge this limitation. On Long3D, completion values (0.057-1.854) suggest the pruning discards tokens relevant to surface completeness.
  • Single architecture dependency: all results build on VGGT/StreamVGGT. Generalization to other vision transformers or attention mechanisms remains untested.
  • Cosine similarity assumption: the claim that key-space similarity is equivalent to attention-based importance is justified intuitively but lacks theoretical proof. Edge cases (e.g., highly non-uniform query distributions) are not examined.
  • Limited diversity in evaluation scenes: Long3D contains only five indoor/campus scenes. Outdoor, dynamic, or adversarial sequences are absent.
