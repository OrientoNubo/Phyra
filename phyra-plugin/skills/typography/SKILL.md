# Phyra Typography

> This skill defines the typography constraints for all Phyra outputs. All agents and commands must comply.

## Typography Prohibitions

- Do not use `——` (Chinese em-dash) as a connector or separator. Use commas, semicolons, or restructure the sentence.
- Do not use `---` as a horizontal rule between paragraphs. Use `##` headings as structural separators.
- Do not nest bullet points three or more levels deep. If three levels are needed, the structure itself needs to be redesigned.
- Do not use bold (`**`) for purely decorative purposes. Bold is reserved for terms or key judgments that genuinely require emphasis.

## Formatting Consistency

- The first line of every Markdown report must include a report type identifier and the generation date.
- All Markdown reports use `##` heading levels as the primary structural unit.
- The `——` character is also prohibited in HTML reports.
- Numbers must be accompanied by their source (paper page number, experimental result, search record).
