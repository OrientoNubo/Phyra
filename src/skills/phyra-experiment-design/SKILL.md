# 實驗設計規範

> 指導 experiment-planner agent 如何設計充分且有說服力的實驗。

## Baseline 選擇標準（至少涵蓋以下三類）

### 1. 同期 SOTA（contemporaneous SOTA）

同時期最強的競爭方法，必須是真實比較，不得選擇弱於自身的 SOTA。

- 選擇原則：必須是同一任務上目前公認的最優結果
- 確認方式：檢查最近 1-2 年的頂會論文和 leaderboard
- 注意事項：不得刻意選擇較弱的「SOTA」來襯托自身方法

### 2. 經典方法（classic baseline）

領域內確立的標桿方法，通常是 3 年以上且被廣泛引用的。

- 選擇原則：該方法在領域內被廣泛認可和引用
- 意義：展示方法相對於經典技術路線的進步幅度
- 注意事項：如果經典方法有已知的改進版本，應同時考慮

### 3. 消融基準（ablated version）

去掉本文核心設計的簡化版，用於驗證設計選擇的必要性。

- 選擇原則：應針對論文的每一個核心貢獻設計對應的消融基準
- 意義：證明每個設計決策都是必要的
- 注意事項：消融基準應盡量只改變一個變量

## 消融實驗設計規範

### 假說驅動

每個消融實驗必須有明確的假說：去掉 X 之後，預期 Y 指標會下降，因為 Z。

- 在實驗前寫下預期結果
- 如果實際結果與預期不符，必須分析原因
- 假說應該基於方法設計的邏輯推導

### 消融粒度

消融不得只做「去掉某模塊」；如果整個模塊都是貢獻，消融粒度要更細。

- 對於複雜模塊，應設計多層次的消融
- 例如：不僅去掉整個 attention module，還要測試不同的 attention 變體
- 粒度應細到能分辨每個設計選擇的影響

### 貢獻量化

消融結果必須能回答「每個設計選擇的貢獻量是多少」。

- 報告每個消融實驗的性能差異（絕對值和相對值）
- 確保所有消融實驗的貢獻量之和能解釋總體性能提升
- 如果存在交互效應，需要額外實驗驗證

## 評測指標選擇原則

### 領域慣用指標

必須使用領域慣用指標。

- 在論文中說明為什麼選擇這些指標
- 如果提出新指標，必須同時報告慣用指標作為對照
- 參考該領域頂會論文的常見指標組合

### 主要指標說明

使用多個指標時，必須說明哪個是主要指標及其理由。

- 主要指標應該最直接地反映任務目標
- 在結果分析中以主要指標為核心展開討論
- 輔助指標用於提供多角度的性能評估

### 禁止 cherry-picking

不得只報告對自己有利的指標子集。

- 所有預先設定的指標都必須完整報告
- 即使某些指標上自己的方法不是最優，也必須如實呈現
- 在分析中誠實討論自己方法的局限性

## 統計顯著性要求

詳見 `~/.phyra/docs/phyra-ai-review-checklist.md` 中各領域的統計慣例部分。

- 根據領域慣例選擇適當的統計檢驗方法
- 報告 p-value 或 confidence interval
- 在結果表中標註統計顯著性

## 實驗充分性自檢清單

在提交論文前，使用以下清單確認實驗設計的完整性：

- [ ] 是否有 held-out 的測試集（而不是把驗證集當測試集）？
- [ ] Baseline 的結果是自己跑的還是直接引用的？如果引用，環境是否一致？
- [ ] 最佳超參數是在測試集還是驗證集上選的？
- [ ] 結果的方差是多少？是否做了多次實驗？
