Overview

Research Topic

本文研究大規模、無姿態(unposed)影像集合下的離線 feed-forward 三維場景重建,目標是同時預測各影像的相機外參、內參與密集深度圖。代表性方法 VGGT 透過 Transformer 的 global self-attention 在 token 空間做跨視圖融合,雖精度高,但計算與記憶體隨輸入影像數 $n$ 呈 $O(n^2)$ 增長,導致無法擴展到上千張遊客式影像或地標重建。作者觀察到此瓶頸源自 attention 中變動長度的 Key-Value (KV) 場景表示,因此提出 VGG-T3:在每個 global attention 層,以 test-time training 將 KV 對應壓縮為固定大小的 MLP 權重(快權重),前向時改為對 query 套用 MLP。如此將複雜度降為 $O(n)$,同時保留全域場景聚合能力,並支援單卡離線、CPU offload 與多卡分散式推論,且自然延伸至 feed-forward 視覺定位。

Core Argument

作者主張現有離線 feed-forward 多視圖重建模型(如 VGGT、Fast3R、π3)無法 scale 的「根因」並非 softmax attention 本身的算子設計,而是其所隱含的「變動長度 KV 場景表示」:每張新加入的影像都會延長 KV 序列,導致 attention 在 query 階段必須對全部 token 做點積,複雜度因而為 $O(n^2)$,且記憶體隨 $n$ 線性增長。FastVGGT、SparseVGGT 之類方法只是壓縮 KV 的常數因子,漸進複雜度仍為平方;線上方法如 CUT3R 雖採固定大小狀態,卻是 auto-regressive 且容易漂移,難以處理無序大型影像集。基於此,作者主張:若能把 KV 場景表示由「變動長度的隱式快取」改為「固定大小的隱式狀態」,複雜度才會本質上從 $O(n^2)$ 降為 $O(n)$。其方案是借鏡 DeepSDF 的固定 latent + test-time 最佳化思維,並結合 Sun et al. 的 TTT 層,在每個 global attention 層用一個小型 MLP 透過自監督目標 $L(T_\theta(k), v)$ 來「蒸餾」KV 對應;查詢時只需 $T_\theta(q)$,既保留全域聚合能力又得到線性複雜度。為了讓預訓練 VGGT 權重能順利 linearize,他們以 L2 normalization 取代 LayerNorm 以加速 TTT 收斂,並在 Value 端加入 ShortConv2D 進行 2D 空間混合,避免 K→V 因同源線性投影產生平凡解;同時透過增加 TTT 最佳化步數實現序列長度的 test-time scaling。這套設計使最佳化目標可分解為 mini-batch 梯度之和,從而支援 CPU offload 與多卡分散式重建,並讓凍結的 MLP 自然成為可被新影像查詢的場景表示,統一了 mapping 與視覺定位。

Highlights

  • 將 VGGT 的 global softmax attention 改寫為 TTT-based 的 MLP 壓縮層,讓 KV 場景表示從可變長度變為固定大小,理論複雜度由 $O(n^2)$ 直接降為 $O(n)$(§3.2,Fig. 2b)。
  • 在 1000 張影像的 1-minute time budget 下完成大型地標重建,VGGT 需 ≥11 分鐘、FastVGGT 需 >4 分鐘,VGG-T3 僅 58 秒,單卡達到 11.6× 加速(Fig. 1b、Fig. 4)。
  • 配合 mini-batch 梯度分片,可在 4 GPU 上 48.5 秒完成 2k 張影像重建,相對 VGGT 的 27 分鐘為 33× 加速(Tab. 4)。
  • 在 pointmap estimation 上對另一個 $O(n)$ 方法 TTT3R 形成壓制,DTU/ETH3D/NRGBD-D 的 Chamfer Distance 降為 TTT3R 的 1/2 至 1/2.5,且 DTU 上甚至優於 VGGT 等 $O(n^2)$ baseline(Tab. 1)。
  • Linearization 從 VGGT 預訓練權重啟動,訓練成本僅為從零訓練 VGGT 的 ≈12%(8× A100-80GB、100k steps),為 post-training linearization 路線提供可行範例(§4 implementation details)。
  • 透過將 Value 端加入 ShortConv2D 打破 $K, V$ 同源線性投影的平凡解,把 mAA(30) 從 72.16 提升到 74.14,逼近 softmax 上限 76.33(Tab. 6)。
  • 提出以「增加 TTT 最佳化步數」作為 test-time scaling 機制,從 1 步增到 2 步即把 1k 影像的 Chamfer Distance 從 ≈0.15 拉回 ≈0.05,跨 50× 序列長度仍近乎常數(Fig. 3、Fig. 5)。
  • 由 $\sum_i d/d\theta\, L(k_i,v_i)$ 可分解為 mini-batch 梯度之和,無需 ring attention 即可直接在 DDP 上分散式推論,並支援 CPU offload 在單卡跑 2k 影像(§3.3、Tab. 4)。
  • 將 mapping 與視覺定位統一在同一模型:凍結 TTT MLP 後以 query image 之 $q$ 套用 $T_\theta(q)$,在 7Scenes 與 Wayspots 上的 10cm/10° localization 命中率分別優於 TTT3R 達 +6.1pp 與 +12.7pp(Tab. 5)。
  • 把 LayerNorm 換成 L2 normalization,顯著加速 TTT 從預訓練權重收斂,使 post-training linearization 比 from-scratch TTT 訓練優異(Tab. 6:from-scratch CD 0.262 vs ours 0.066)(§3.2)。

3. Section Walkthrough

3.1 Title and Abstract

(180 words)

論文標題「VGG-T3: Offline Feed-Forward 3D Reconstruction at Scale」直接點出三個關鍵定位：以 VGGT 為基底、屬於 offline feed-forward 類別、訴求是「at Scale」的可擴展性。Abstract 隨即把問題鎖定在現有 offline feed-forward 方法的核心痛點：計算與記憶體需求隨輸入影像數量呈 quadratic 成長。作者提出的關鍵洞察是，這個 bottleneck 來自於 global self-attention 中 variable-length 的 KV space 表示，並主張可以用 test-time training 把這個 KV space distill 成 fixed-size MLP。

接著 Abstract 直接給出量化承諾：VGG-T3 (Visual Geometry Grounded Test Time Training) 對輸入影像數量呈 linear scaling，可在 54 秒內重建 1k 張影像的 collection，相對於 softmax-attention baselines 達到 11.6× 加速。並指出由於保留了 global scene aggregation 能力，point map 重建誤差大幅優於其他 linear-time 方法。最後預告一個延伸能力：可用 unseen images 查詢 scene representation，達成 visual localization。

這段敘事建立了論文後續所有討論的核心主軸：「KV space 是 quadratic bottleneck 的根源，把它換成 fixed-size MLP 就能 unlock scalability 並順帶得到 query-able representation」，為 §1 Introduction 鋪陳問題框架。

3.2 Introduction

(620 words)

Introduction 採用「現況→瓶頸根因→解法直覺→延伸應用→貢獻」的五段式結構。

首先是 status quo：基於 Transformer 的 feed-forward 方法（VGGT、Fast3R、π3 等）在精度上已可媲美傳統 SfM pipeline (COLMAP、GLOMAP)，且對 rapid camera motion 與 low visual overlap 等挑戰更 robust。但作者立刻點出代價：computational 與 memory 需求對輸入影像數的 scaling 很糟。

第二段把瓶頸根因明確化：問題來自 global self-attention layer 中 KV space 作為「dense, variable-length scene representation」的設計。要從這個 latent representation 萃取 3D 屬性必須執行 global softmax attention，而這個操作對輸入影像數呈 quadratic 複雜度。作者點名近期 sparse attention (SparseVGGT) 與 token merging (FastVGGT) 雖能壓縮表示長度，但並未改變 underlying quadratic scaling。

第三段「Compress your KV」是論文的核心 insight：相對於 variable-length 表示，存在另一條路線——以 fixed-size implicit representation 表達場景（如 DeepSDF）。作者把 DeepSDF 的精神搬到 multi-view feed-forward 設定：保留 pre-trained encoder/decoder，但將 global attention layer 的 KV mapping 改用 fast weights 的 MLP，藉由 test-time optimization 在 token space 學習這個 mapping。如此一來，querying KV space 就退化為「對 input tokens 套用 MLP」，操作對影像數呈 linear。

第四段「Large-scale feed-forward reconstruction」推導出此設計帶來的工程紅利：由於 TTT loss 是 token-level local loss 的加總，gradient 可以 minibatch 拆分。這支援兩種 inference strategy：(i) 單 GPU 透過 CPU offloading 處理大型 collection，(ii) 跨多 GPU 透過 token sharding 做 distributed inference。具體成效是 2k 影像可在 48.5 秒處理完畢，相較 VGGT 的 27 分鐘有 33× 提升。

第五段「Visual localization」拋出一個延伸 capability：optimized MLP 可視為 scene 的 compressed representation，凍結後以 novel query view 詢問即可進行 feed-forward visual localization，而傳統上 reconstruction 與 localization 是兩條獨立 pipeline。

最後條列四項 contribution：(i) 提出 linear-scaling 的 offline feed-forward 重建模型；(ii) 證明可將 variable-length KV scene representation 轉為 fixed-dimensional implicit state；(iii) 同時支援 single-GPU large-image-set inference 與 efficient distributed inference；(iv) 展示 mapping 與 visual localization 統一於單一模型的 proof-of-concept。

整段 Introduction 的敘事邏輯是先確立「為什麼要 linear」、再給出「如何 linear 而不犧牲 global aggregation」的具體機制，最後 framing 為通用 representation 改造，為 §2 Related Work 與 §3 Method 同時鋪設舞台。

3.3 Related Work / Preliminaries

(1150 words)

§2 Related Work 先把 3D reconstruction 領域分為四條主軸：(1) 古典 SfM pipeline (Bundler、COLMAP、GLOMAP)，特色是多階段、需要充分 visual overlap；(2) feed-forward pairwise/multi-view models (DUSt3R、MASt3R、VGGT、Fast3R、π3、Light3R-SfM、FLARE)，這類方法的共通骨架是 tokenizer + multi-view aggregator + decoder，差異在於如何做 multi-view fusion；(3) large-scale 重建分支，又分為 chunk-based (Slam3R、VGGT-SLAM、VGGT-Long) 與 attention-optimization 兩派 (FastVGGT 用 token merging、SparseVGGT 用 block-sparse)，作者強調這些做法只是 reduce constant factor，asymptotic 仍是 quadratic；(4) online 與 streaming 方法，其中 fixed-size implicit memory (CUT3R、Must3R) 與 explicit spatial memory (Point3R、Long3R、MapAnything) 已經是 linear-time，但屬於 auto-regressive。

接著作者點名 concurrent work TTT3R——同樣使用 test-time training，但建構於 CUT3R 的 fixed-size memory 之上，本質仍是 auto-regressive。VGG-T3 則是 global (offline)，因此精度顯著更高。再來討論 linear attention 的整體研究脈絡：linear attention、SSM (S4、H3、Hyena、Mamba) 與 TTT 之間的等價性，並引述 LLM 領域的 post-training linearization 趨勢，把自己的工作 framing 為「將 post-training linearization 延伸到 multi-view reconstruction」。最後 visual localization 段落鋪陳 SCR 路線 (ACE、ACEZero) 與 feature matching + PnP 路線，作為 §4.3 的對照背景。

§3.1 Preliminaries 為 §3.2 的方法做雙重鋪陳。先回顧 VGGT：image tokenizer 把每張影像轉為 token sequence $x_i$，再交給交替的 image-wise self-attention 與 global self-attention blocks 處理。每個 attention block 中 token 經由 learned linear projection $W_q, W_k, W_v$ 與 layer norms $\text{LN}_q, \text{LN}_k$ 得到 $q_i, k_i, v_i$，softmax attention 為

$o_i = \sum_j \mathrm{softmax}_j\!\left(\frac{q_i^T k_j}{\sqrt{d}}\right) v_j.$

prediction heads 直接從 output tokens 解出 per-image depth、camera poses、intrinsics。作者特別點出 global self-attention 是 multi-view understanding 不可或缺的部分，但也是 quadratic 複雜度的源頭。

接著回顧 test-time training 的數學形式：從 Sun et al. 的觀點，attention 可以被重新詮釋為一個 fast weights $\theta$ 的最佳化問題，

$$\arg\min_{\theta} \sum_i L_t\big(T_\theta(k_i) - v_i\big), \quad o_i = T_\theta(q_i),$$

亦即 $T_\theta$ 學習 $K \to V$ 的 mapping，再用 query $q_i$ 取回對應 value，類比於 softmax 用 QK 相似度檢索 V 的行為。關鍵性質在於：這兩個操作對 sequence length 都是 linear 的。

整體 Related Work 與 Preliminaries 的故事邏輯是：(a) 確立 quadratic 是 offline feed-forward 路線的根本病灶；(b) 既有所有解法 (chunk、token merging、sparse、auto-regressive memory) 都各有妥協；(c) TTT 提供一套「把 attention 轉為 fixed-size optimization variable」的數學工具，且已在 LLM、video、view synthesis 等領域 validated。這恰好為 §3.2 提出的「把 VGGT global attention 換成 TTT-based MLP」做出技術可行性與差異化定位的雙重背書。

3.4 Method (overview narrative)

(1700 words)

§3 Method 的整體敘事走「核心問題 → 概念性 substitution → 從 pre-trained 模型 linearize 的工程細節 → expressivity 修補 → test-time scaling → 大規模推論的工程含義 → query-able representation」的長弧線。

§3.2 開場以一個尖銳的問題框定整章：「Can We Fit Rome into MLPs?」作者把 multi-view scalability 問題歸結為 scene representation 的選擇，並重申 VGGT quadratic 的根因是 KV scene representation 為 variable-length。Overview 給出方法骨架：每個 global attention layer 變成兩階段——(1) Update：將 input tokens 投影為 $q, k, v$，用 TTT 將 KV mapping 壓縮進 fixed-size MLP 的 fast weights $\theta$；(2) Apply：套用該 MLP 到 queries 取得 updated tokens，傳給下一層。decoder 仍只在最後一層 transformer 之後執行。

接著 Linearizing the pre-trained model 段落把工程細節攤開：作者希望從 VGGT 的 pre-trained $W_q, W_k, W_v$ 初始化以省成本，但發現 naive 替換收斂極慢。診斷出原因是 LayerNorm 中可學習參數會扭曲 MLP 要學的 input space；移除 LN、改用 L2 normalization 後 convergence 顯著改善。同時也驗證一個訓練策略：先以 softmax attention 訓練、再做 post-training linearization，比起 from-scratch TTT 表現更佳。

Non-linear spatial mixing 是 method 中最關鍵的 expressivity 修補。作者點出 TTT 目標 $K \to V$ 因為 $K = W_k x$ 與 $V = W_v x$ 都源自同一 token 的 linear projection，存在 trivial solution $V = W_v W_k^{-1} K$，使得 MLP 學不到有意義的場景關聯。為了打破這個 dependency，作者引入 ShortConv2D：把 1D token sequence 重塑回 2D image grid $(N, H/p, W/p, d)$、套用單層 2D convolution 聚合鄰域資訊得到 context-aware target $V'$、再 flatten 回 1D 後做 TTT optimization，把目標改為 $K \to V'$。直覺是 V 端帶有 local spatial context 而 K 端沒有，迫使 MLP 學習更 robust 的 geometric scene representation。

Test-time scaling 處理 sequence length generalization 問題：訓練時 collection size 上限為 24，但部署時要面對 1k+ images。作者觀察到單一 optimizer step 在 1k images 上會導致 reconstruction error 比 100 images 上高約 5×，並透過 Fig. 3a 顯示 best optimizer step 隨 collection 規模上升而往後移。透過增加 TTT 內部 optimizer steps（最後 default 為 2 steps），可達到接近 constant 的 scaling，這被作者 framing 為一種 test-time scaling via additional computation。

§3.3 Large-scale Reconstruction 接著把這個 representation change 的工程含義系統化。複雜度上，從 $O(n^2)$ 降為 $O(n)$。Flexible inference 部分仔細推導出關鍵性質：

$$\frac{dL_{\text{total}}}{d\theta} = \sum_i \frac{d}{d\theta} L(k_i, v_i) = \sum_s \left(\sum_{i \in s} \frac{d}{d\theta} L(k_i, v_i)\right),$$

也就是 total gradient 可被任意 minibatch 拆分。這 unlock 兩個場景：(a) 單 GPU 上 off-load minibatch 到 host memory 處理任意大 collection；(b) 多 GPU 上 shard image tokens、各自算 local gradient，再用 small-size all-to-all 同步 MLP weights。對比之下 softmax attention 必須把所有 $q, k, v$ 同時放在 GPU memory，遇到大 collection 必然 OOM。

最後 Query-able reconstruction & visual localization 把 representation 屬性轉成 capability：完成 mapping 後凍結 fast weights，對 query image 執行單張 forward pass，但在 global attention layer 只把凍結的 MLP apply 到 query features，不更新 $\theta$。這實際上把 model 轉為 single-image transformer，並自然延伸到 visual localization。

整章敘事為 §4 Experiments 設好三條驗證軸線：standard benchmarks 上的精度持平、large-scale regime 上的 scaling、visual localization 的 capability 展示，每一條都對應方法章中明確的 design decision。

3.5 Experiments (overview narrative)

(700 words)

§4 Experiments 開頭以四個子節的 roadmap framing 整章：standard benchmarks (§4.1) 驗證在傳統規模下精度沒有崩；large-scale regime (§4.2) 驗證 linear scaling 在實務上兌現；visual localization (§4.3) 把 method 中宣稱的 query-able representation 落地；ablations (§4.4) 拆解 design choice 的個別貢獻。

Implementation details 一次交代清楚：以 public VGGT checkpoint 為起點，把所有 global attention layer 替換為 TTT layer；fast weights MLP 用 SwiGLU 結構，optimizer 為 Muon，loss 為 dot-product loss $L_t(T_\theta(k_i), v_i) = T_\theta(k_i)^T v_i$，並在 V 上加 $3\times 3$ ShortConv2D。訓練只 fine-tune global attention layer，凍結其餘 VGGT 參數，跑 100k steps 在 8 張 A100-80GB 上，成本約為從零訓練 VGGT 的 12%。

Baselines 分兩類：offline 端以 VGGT 為精度上限、配上 FastVGGT 與 SparseVGGT 兩個 quadratic 但已最佳化的變體；online 端則對標 concurrent 的 TTT3R，後者是同樣使用 TTT 但建構於 auto-regressive memory 之上的 linear-complexity 方法。這個 baseline 設計直接對映 §2 中 framing 出的兩個競爭路線。

§4.1 Standard Benchmarks 涵蓋三個 downstream task：pointmap estimation (NRGBD、7scenes、DTU、ETH3D，metric 為 Chamfer Distance 與 Normal Consistency)、video depth (Bonn、KITTI、Sintel)、camera pose estimation (TUM-RGBD、ScanNet、Sintel)。整體 narrative 是：相對於唯一的 $O(n)$ baseline TTT3R，VGG-T3 在 pointmap 與 video depth 多數 split 上有顯著優勢；相對於 $O(n^2)$ baselines 則保持競爭力，甚至在 DTU 上反超。Camera pose 是相對弱項，作者明確 attribute 給 VGGT 的 camera token 異質結構，難以被 MLP memorize，並把它列為 future direction。同時凸顯一個方法層面的優勢：VGG-T3 自然支援 unordered input，而 TTT3R 在 unordered 設定下大幅退化。

§4.2 Large-Scale 3D Reconstruction 是論文最有 punch 的實驗區塊。在 7scenes 上以 100/500/1k 張 image 量測 runtime vs Chamfer distance，VGG-T3 在 1k 張時 58 秒完成、相對 VGGT 的 11 分鐘達到 11.6× 加速、相對 FastVGGT 的 4 分鐘有 4.3× 加速；對 TTT3R 雖然 runtime 接近但 reconstruction 精度顯著更高，且不會隨 sequence 長度 degrade。Distributed inference 表 (Tab. 4) 進一步顯示 VGG-T3 用 plain DDP 即可線性加速 (1500 影像：1/2/4 GPU 為 173.1/56.8/29.7 秒)，而 VGGT 需要 ring attention 等 context-parallel 工程才能勉強跑 2k images。

§4.3 Feed-forward Visual Localization 在 7scenes 與 Wayspots 上對標 TTT3R，使用 rotation/translation error 與 10/20cm-10/20° accuracy。VGG-T3 在兩個 dataset 上都贏，Wayspots 的差距尤其明顯，佐證 MLP-based scene representation 的 query 能力。作者也誠實 caveat：與 explicit-mapping 的 SOTA visual localization (如 Reloc3R) 仍有差距，目標只是展示 feed-forward visual localization 不依賴顯式 mapping 的可行性。

§4.4 Ablations 在 224×224 ScanNet++ small-scale 設定下，逐一驗證 design：from-scratch TTT 卡在 local optimum、必須從 softmax-attention pretrain linearize；T2R、LoLCats 兩個 LLM 領域的 linearization baseline 都明顯遜於本作；最後加入 ShortConv2D 進一步縮小與 softmax attention 的差距。整體實驗章節的故事弧線是：standard benchmarks 證明沒退步、large-scale 證明真值得換 representation、localization 展示 bonus capability、ablation 把功勞精準歸因到 design choices。

3.6 Conclusion / Limitations / Future Work

(160 words)

§5 Conclusion 篇幅極短但定位清晰，純粹收束全文 thesis：作者重申 contribution 為「scalable feed-forward 3D reconstruction that gracefully scales with the number of input views」，核心機制是用 test-time optimization 學 $K \to V$ mapping 取代 softmax attention 的 KV querying。再以一組量化 punchline 收尾——VGG-T3 在 1k images 上 11.6× 加速、2k images 上達 33× 加速，同時在 pointmap 與 video depth estimation 上大幅優於其他 linear-time 方法。

Limitations 段落作者誠實坦承 trade-off：linear-time MLP scene representation 雖然成功保留 scalability 與 global aggregation 屬性，並大勝 auto-regressive 的 online 方法，但相對於 softmax attention 仍存在精度差距，特別是在 wide-baseline 場景。作者把這個現象 frame 成一個 open research question：如何 reconcile fixed-expressivity MLP scene representation 與 quadratic attention 的高精度。

雖然論文未獨立列出 Future Work 段，但結尾一句明確指出後續方向——尋找介於 fixed-size MLP 與 full softmax attention 之間、能 match softmax 精度卻保持 linear scaling 的更具表達能力 representation。Appendix D 補充的 Waymo wide-baseline 失敗案例，也呼應這個 future direction，並暗示自適應依場景複雜度調整 computation 的設計可能是一條路徑。

Introduction

(620 words)

Introduction 採用「現況→瓶頸根因→解法直覺→延伸應用→貢獻」的五段式結構。

首先是 status quo：基於 Transformer 的 feed-forward 方法（VGGT、Fast3R、π3 等）在精度上已可媲美傳統 SfM pipeline (COLMAP、GLOMAP)，且對 rapid camera motion 與 low visual overlap 等挑戰更 robust。但作者立刻點出代價：computational 與 memory 需求對輸入影像數的 scaling 很糟。

第二段把瓶頸根因明確化：問題來自 global self-attention layer 中 KV space 作為「dense, variable-length scene representation」的設計。要從這個 latent representation 萃取 3D 屬性必須執行 global softmax attention，而這個操作對輸入影像數呈 quadratic 複雜度。作者點名近期 sparse attention (SparseVGGT) 與 token merging (FastVGGT) 雖能壓縮表示長度，但並未改變 underlying quadratic scaling。

第三段「Compress your KV」是論文的核心 insight：相對於 variable-length 表示，存在另一條路線——以 fixed-size implicit representation 表達場景（如 DeepSDF）。作者把 DeepSDF 的精神搬到 multi-view feed-forward 設定：保留 pre-trained encoder/decoder，但將 global attention layer 的 KV mapping 改用 fast weights 的 MLP，藉由 test-time optimization 在 token space 學習這個 mapping。如此一來，querying KV space 就退化為「對 input tokens 套用 MLP」，操作對影像數呈 linear。

第四段「Large-scale feed-forward reconstruction」推導出此設計帶來的工程紅利：由於 TTT loss 是 token-level local loss 的加總，gradient 可以 minibatch 拆分。這支援兩種 inference strategy：(i) 單 GPU 透過 CPU offloading 處理大型 collection，(ii) 跨多 GPU 透過 token sharding 做 distributed inference。具體成效是 2k 影像可在 48.5 秒處理完畢，相較 VGGT 的 27 分鐘有 33× 提升。

第五段「Visual localization」拋出一個延伸 capability：optimized MLP 可視為 scene 的 compressed representation，凍結後以 novel query view 詢問即可進行 feed-forward visual localization，而傳統上 reconstruction 與 localization 是兩條獨立 pipeline。

最後條列四項 contribution：(i) 提出 linear-scaling 的 offline feed-forward 重建模型；(ii) 證明可將 variable-length KV scene representation 轉為 fixed-dimensional implicit state；(iii) 同時支援 single-GPU large-image-set inference 與 efficient distributed inference；(iv) 展示 mapping 與 visual localization 統一於單一模型的 proof-of-concept。

整段 Introduction 的敘事邏輯是先確立「為什麼要 linear」、再給出「如何 linear 而不犧牲 global aggregation」的具體機制，最後 framing 為通用 representation 改造，為 §2 Related Work 與 §3 Method 同時鋪設舞台。

Related Work

(1150 words)

§2 Related Work 先把 3D reconstruction 領域分為四條主軸：(1) 古典 SfM pipeline (Bundler、COLMAP、GLOMAP)，特色是多階段、需要充分 visual overlap；(2) feed-forward pairwise/multi-view models (DUSt3R、MASt3R、VGGT、Fast3R、π3、Light3R-SfM、FLARE)，這類方法的共通骨架是 tokenizer + multi-view aggregator + decoder，差異在於如何做 multi-view fusion；(3) large-scale 重建分支，又分為 chunk-based (Slam3R、VGGT-SLAM、VGGT-Long) 與 attention-optimization 兩派 (FastVGGT 用 token merging、SparseVGGT 用 block-sparse)，作者強調這些做法只是 reduce constant factor，asymptotic 仍是 quadratic；(4) online 與 streaming 方法，其中 fixed-size implicit memory (CUT3R、Must3R) 與 explicit spatial memory (Point3R、Long3R、MapAnything) 已經是 linear-time，但屬於 auto-regressive。

接著作者點名 concurrent work TTT3R——同樣使用 test-time training，但建構於 CUT3R 的 fixed-size memory 之上，本質仍是 auto-regressive。VGG-T3 則是 global (offline)，因此精度顯著更高。再來討論 linear attention 的整體研究脈絡：linear attention、SSM (S4、H3、Hyena、Mamba) 與 TTT 之間的等價性，並引述 LLM 領域的 post-training linearization 趨勢，把自己的工作 framing 為「將 post-training linearization 延伸到 multi-view reconstruction」。最後 visual localization 段落鋪陳 SCR 路線 (ACE、ACEZero) 與 feature matching + PnP 路線，作為 §4.3 的對照背景。

§3.1 Preliminaries 為 §3.2 的方法做雙重鋪陳。先回顧 VGGT：image tokenizer 把每張影像轉為 token sequence $x_i$，再交給交替的 image-wise self-attention 與 global self-attention blocks 處理。每個 attention block 中 token 經由 learned linear projection $W_q, W_k, W_v$ 與 layer norms $\text{LN}_q, \text{LN}_k$ 得到 $q_i, k_i, v_i$，softmax attention 為

$o_i = \sum_j \mathrm{softmax}_j\!\left(\frac{q_i^T k_j}{\sqrt{d}}\right) v_j.$

prediction heads 直接從 output tokens 解出 per-image depth、camera poses、intrinsics。作者特別點出 global self-attention 是 multi-view understanding 不可或缺的部分，但也是 quadratic 複雜度的源頭。

接著回顧 test-time training 的數學形式：從 Sun et al. 的觀點，attention 可以被重新詮釋為一個 fast weights $\theta$ 的最佳化問題，

$$\arg\min_{\theta} \sum_i L_t\big(T_\theta(k_i) - v_i\big), \quad o_i = T_\theta(q_i),$$

亦即 $T_\theta$ 學習 $K \to V$ 的 mapping，再用 query $q_i$ 取回對應 value，類比於 softmax 用 QK 相似度檢索 V 的行為。關鍵性質在於：這兩個操作對 sequence length 都是 linear 的。

整體 Related Work 與 Preliminaries 的故事邏輯是：(a) 確立 quadratic 是 offline feed-forward 路線的根本病灶；(b) 既有所有解法 (chunk、token merging、sparse、auto-regressive memory) 都各有妥協；(c) TTT 提供一套「把 attention 轉為 fixed-size optimization variable」的數學工具，且已在 LLM、video、view synthesis 等領域 validated。這恰好為 §3.2 提出的「把 VGGT global attention 換成 TTT-based MLP」做出技術可行性與差異化定位的雙重背書。

Method — Overview

Narrative

VGG-T3 將 VGGT 的 global self-attention 替換為 test-time training (TTT) 層,核心想法是在每個 global attention block 把可變長度的 KV cache「蒸餾」成固定大小的 MLP 權重(fast weights $\theta$),前向傳遞時改為對 query 套用該 MLP。形式上,給定影像 token 的 $q_i, k_i, v_i \in \mathbb{R}^d$,標準 softmax attention 為 $o_i = \sum_j \mathrm{softmax}_j(q_i^T k_j / \sqrt d) v_j$,複雜度 $O(n^2)$;TTT 則改為先解 $\arg\min_\theta \sum_i L_t(T_\theta(k_i), v_i)$,再以 $o_i = T_\theta(q_i)$ 產生輸出 (Eq. 3、Eq. 4),兩個步驟對序列長度皆為 $O(n)$。每層 global attention 因此分成兩個階段:Update 用 TTT 把 KV 對應壓縮為 MLP 權重(視為 fast weights,訓練與推論時同樣最佳化);Apply 將最佳化後的 $T_\theta$ 套到該層 queries 得到下一層的 token,只有在最後一層才解碼出 depth、相機外參與內參。

為了讓預訓練 VGGT 權重能順利 linearize,本文先將 QK 端的 LayerNorm 換成 L2 normalization,因為 LN 中可學的縮放/偏移參數會扭曲 MLP 在測試時要學習的輸入空間,實驗證明能加速 TTT 收斂(§3.2,p.5)。同時,由於 $K = W_k x$ 與 $V = W_v x$ 來自同一 token 的線性投影,$L_t$ 容易塌成 $V = W_v W_k^{-1} K$ 的平凡解;因此在 V 端額外加入 ShortConv2D (重塑成 $(N, H/p, W/p, d)$ 後做 $3\times 3$ 2D 卷積,再展平),強迫 TTT 學習 $K \to V'$,其中 $V'$ 帶有局部空間鄰域資訊,提供更強的自監督目標。

實作上採用 LaCT 的 SwiGLU MLP 作為 $T_\theta$、Muon 作為內層最佳化器、dot-product loss $L_t(T_\theta(k_i), v_i) = T_\theta(k_i)^T v_i$。inner hidden 維度為 $4d$($d=1024$),只在 global attention 的 QKV/output 投影與 TTT 模組訓練,VGGT 其餘參數凍結。Test-time scaling 透過增加 TTT 最佳化步數(預設 2 步)讓固定大小 MLP 能容納更大規模的場景(§3.2 與 Fig. 3)。最重要的是,$L_t$ 為 token 上的求和,梯度也可拆成 mini-batch 之和 (Eq. 5),因此在多 GPU 上以 DDP + all-to-all 同步 MLP 權重即可分散式推論,在單 GPU 上則可將 mini-batch offload 到 host memory,逐批計算梯度後同步,使單卡能處理上千張影像(§3.3)。

Method — 5.3.1 Image Tokenizer (frozen, from VGGT)

Function: 將每張輸入影像轉換為 patch token 序列,並在每張影像內部做 image-wise self-attention,提供進入 global block 的 token 表示。

Input:

  • Name: $I = \{I_i\}_{i=1}^N$
  • Shape: [N, 3, H, W]
  • Source: 原始多視圖影像集合

Output:

  • Name: $x$
  • Shape: [N·P, d],其中 $P = (H/p)(W/p)$、$p=14$、$d=1024$
  • Consumer: 第一層 global attention block 的 QKV 投影

Processing:

每張影像被切成 $H/p \times W/p$ 個 patch,以 ViT-style patch embedding 投影到 $d=1024$ 維;在每個 alternating block 中先執行 image-wise self-attention(僅限同一張影像內 token),其後再進入 global attention(本文替換之處)。輸出 token 會與 camera token、register token 一起進入下一個 block,但僅 image patch token 會經過 ShortConv2D 與 TTT update;camera/register token 直接 pass through(Implementation Details, p.10)。

Key Formulas:

The paper does not specify the exact tokenizer formula; 本模組沿用 VGGT 設計。

Implementation Details:

Tokenizer 與 image-wise self-attention 的所有參數在訓練 VGG-T3 時皆凍結,只 fine-tune 全域 attention 層的 QKV/output 投影與新加入的 TTT 模組(§A, p.10)。輸入影像的長邊 resize 至 518,aspect ratio 從 $[0.5, 2.0]$ 均勻取樣;使用 patch size $p=14$,因此 VGGT 訓練時的最大 token 數為 $N_T = 24 \times (518/14)^2 = 32{,}856$(§B, p.11)。

Method — 5.3.2 QKV Projection with L2 Normalization

Function: 從 token 投影 query/key/value,並以 L2 normalization 取代原 VGGT 的 LayerNorm,避免可學參數扭曲 TTT 要學習的輸入空間,加速從預訓練權重收斂。

Input:

  • Name: $x$
  • Shape: [N·P, d]($d=1024$)
  • Source: 上一層的 image-wise self-attention 輸出

Output:

  • Name: $(q, k, v)$
  • Shape: 每個皆為 [N·P, d](多 head 時為 [N·P, h, d_h],$h \cdot d_h = d$)
  • Consumer: $v$ 進入 ShortConv2D;$k, v'$ 進入 TTT update;$q$ 進入 Apply step

Processing:

對輸入 token $x_i$ 計算 $q_i = \text{L2}(W_q x_i)$、$k_i = \text{L2}(W_k x_i)$、$v_i = W_v x_i$。L2 normalization 即 $\text{L2}(u) = u / \lVert u \rVert_2$,沿特徵維度做單位化。原 VGGT 使用 LayerNorm 對 QK 做 normalization 以穩定 softmax attention(QK-norm, Eq. 1);改用 L2 後不再含可學的 affine 參數,經驗上能讓 TTT 從預訓練權重快速收斂(§3.2 Linearizing the pre-trained model, p.5)。

Key Formulas:

$$ q_i = \mathrm{L2}(W_q x_i), \quad k_i = \mathrm{L2}(W_k x_i), \quad v_i = W_v x_i $$

Implementation Details:

只 fine-tune $W_q, W_k, W_v$ 與 output projection,以及新增的 TTT 模組;VGGT 其他部分(encoder、image-wise attention、prediction heads)全部凍結(§A, p.10)。$d=1024$ 與 VGGT hidden size 對齊。

Method — 5.3.3 ShortConv2D on Value

Function: 對 V 端 token 在 2D 影像格上做 $3\times 3$ 深度卷積,聚合局部空間鄰域,把 $K \to V$ 的線性可逆關係打破為 $K \to V'$,讓 TTT 目標不再有平凡解,並提供更強的自監督訊號。

Input:

  • Name: $v$
  • Shape: 序列形式 [N·P, d]
  • Source: §5.3.2 輸出

Output:

  • Name: $v'$
  • Shape: [N·P, d]
  • Consumer: §5.3.4 TTT update step 的目標 value

Processing:

(1) Reshape:將 V 從 1D token sequence 轉成 2D image grid,形狀 [N, H/p, W/p, d],$p$ 為 patch size。 (2) Convolve:沿空間維 $H/p \times W/p$ 做 $3\times 3$ ShortConv2D(2D 卷積比語言模型常用的 1D conv 更適合影像結構),產生 context-aware 的 $V'$。 (3) Flatten:再展回 [N·P, d],作為 TTT 目標。

cameras token 與 register token 不參與 ShortConv2D,直接 pass through(§A Inference details, p.10)。卷積核大小 $3\times 3$ 在 ablation 中最佳;$5\times 5$ 並未進一步改善,將 ShortConv2D 同時加在 $K, V$ 反而下降,因為 $K, V$ 共用相同空間混合會重新建立兩者線性依賴(§C Table 9 ShortConv2D ablation, p.12)。

Key Formulas:

The paper does not specify an exact formula beyond the three-step description above; ShortConv2D 為 single-layer 2D convolution。

Implementation Details:

預設 $3\times 3$ filter,僅作用於 image patch token 對應的 V(§A Inference details, p.10);僅在 V 端套用,不對 K 套用。Concurrent 工作 ViT3 也採用了類似的 inner convolution(§3.2 Non-linear spatial mixing, p.5)。

Method — 5.3.4 Test-Time Training (Update)

Function: 在每個 global attention 層,用自監督 loss 將該層所有 KV 對應「蒸餾」成一個固定大小 MLP 的 fast weights $\theta_\ell$,作為該層的隱式場景表示。

Input:

  • Name: $\{(k_i, v'_i)\}_{i=1}^{N \cdot P}$
  • Shape: 每個皆為 [N·P, d]
  • Source: §5.3.2($k$)與 §5.3.3($v'$)

Output:

  • Name: $\theta_\ell$(該層 fast weights)
  • Shape: 與 MLP 參數同形(SwiGLU MLP,輸入/輸出 1024 維,hidden $4 \times 1024 = 4096$);其大小固定、不隨 $N$ 變化
  • Consumer: §5.3.5 Apply step

Processing:

固定大小 MLP $T_\theta : \mathbb{R}^d \to \mathbb{R}^d$ 採 SwiGLU 結構,輸入維度 1024,hidden 維度 $4d=4096$。對該層所有 token 求解

$$ \theta_\ell = \arg\min_\theta \sum_{i=1}^{N \cdot P} L_t\bigl(T_\theta(k_i), v'_i\bigr), \quad L_t(T_\theta(k_i), v'_i) = T_\theta(k_i)^T v'_i $$

inner optimizer 使用 Muon(5 次 Newton–Schulz 迭代,learning rate 0.1),訓練時內部步數預設 1,推論時預設 2(§A Training, p.10)。由於 $L_t$ 對 $\theta$ 的梯度為 token 上之和,可拆成 mini-batch 子和:

$$ \frac{dL_{\text{total}}}{d\theta} = \sum_i \frac{d}{d\theta} L(k_i, v'_i) = \sum_s \Bigl( \sum_{i \in s} \frac{d}{d\theta} L(k_i, v'_i) \Bigr) \quad (\text{Eq. 5}) $$

於是分散式推論時將影像分片到不同 GPU 各自累加梯度後 all-to-all 同步 $\theta$;單 GPU 大規模推論時把每個 mini-batch offload 到 host memory,只保留當前批次在 device,逐批計算梯度後再 offload 回去(§3.3, p.5–6)。

Key Formulas:

$$ \arg\min_\theta \sum_i L_t\bigl(T_\theta(k_i) - v_i\bigr) \quad (\text{Eq. 3, V replaced by } v') $$

$$ \frac{dL_{\text{total}}}{d\theta} = \sum_s \Bigl( \sum_{i \in s} \frac{d}{d\theta} L(k_i, v_i) \Bigr) \quad (\text{Eq. 5}) $$

Implementation Details:

SwiGLU MLP 的 input/output 維度 $=1024$,hidden $=4096$(§A, p.10)。Inner optimizer 為 Muon、lr $=0.1$、Newton–Schulz 迭代 5 次,訓練時 1 步、推論時預設 2 步;序列越長(out-of-distribution)時提高內部步數能近似常數誤差縮放(§3.2 Test-time scaling, Fig. 3, p.5;§C Number of optimizer steps, Fig. 5, p.11)。Loss 採 dot-product 形式 $T_\theta(k_i)^T v'_i$ 而非平方差(§4 Implementation details, p.6)。

Method — 5.3.5 Test-Time Training (Apply) and Output Projection

Function: 用最佳化後的 fast weights $\theta_\ell$ 對 query 做前向,得到該層替代 softmax attention 的輸出,再經 output projection 與後續 image-wise block,直到最後一層才解碼出深度、相機姿態與內參。

Input:

  • Name: $q$, $\theta_\ell$
  • Shape: [N·P, d],$\theta_\ell$ 為固定大小 MLP 參數
  • Source: §5.3.2($q$)與 §5.3.4($\theta_\ell$)

Output:

  • Name: $o$(該層輸出 token);最終為 $\{P_i, K_i, X_i\}$
  • Shape: $o$ 為 [N·P, d];$P_i$ 為 $(R_i \in SE(3), T_i \in \mathbb{R}^3)$、$K_i \in \mathbb{R}^{3\times 3}$、$X_i \in \mathbb{R}^{H \times W}$
  • Consumer: 下一個 alternating block;最後一層後接 prediction heads

Processing:

對該層所有 query 計算 $o_i = T_{\theta_\ell}(q_i)$(Eq. 4),整體計算為 $O(N \cdot P)$,無須對全部 KV 做點積。然後接原 VGGT 的 output projection、residual 與 image-wise self-attention,進入下一層。Visual localization 設定下,對 mapping 影像跑完所有 update 後保留 $\theta_\ell$ 不再更新,query 影像走過模型時只執行 apply step,以 frozen $\theta_\ell$ 取得 token,等價於把模型當成 single-image transformer 處理新影像(§3.3 Query-able reconstruction & visual localization, p.6)。最終由 VGGT 三組 prediction heads 解碼出 per-view depth $X_i$、相機外參 $P_i = (R_i, T_i)$、pinhole intrinsics $K_i$。

Key Formulas:

$o_i = T_\theta(q_i) \quad (\text{Eq. 4})$

Implementation Details:

實際輸出 pointmap 時,作者使用 VGGT 的 global pointmap head 直接輸出,而非 unproject(depth, pose),因為 TTT 線性化後相機姿態較不準確,unproject 會放大誤差(§A Inference details, p.10)。Visual localization 時,query 影像的 camera token 仍會與 mapping 影像的 camera token 一起進入 camera head 的 softmax attention,再解出相機參數,因為 VGGT camera head 需要全部 camera token(§A Inference details, p.10)。

Settings

訓練從公開的 VGGT checkpoint 出發,將所有 global attention layer 替換為 TTT layer,並只 fine-tune QKV 投影、output projection 與新增的 TTT 模組參數,encoder、per-image attention、prediction heads 全部 frozen(Sec. A)。硬體為 8 張 NVIDIA A100-80GB GPU,訓練 100k steps,作者宣稱整體訓練成本約為 VGGT from-scratch 的 12%。

採用 adaptive batch size:每個 collection 含 2–24 張影像,單 GPU 上影像總數維持約 48 張;影像長邊 resize 至 518,aspect ratio 從 $[0.5, 2.0]$ 均勻取樣,並對每張影像獨立做 color jitter。Image collection sampling 採 greedy 方式,先隨機抽一張,再從與目前集合中任一影像 co-visibility $> 0.3$ 的候選集均勻抽取,co-visibility matrix 透過 depth consistency check 預先計算(Sec. A)。

Outer optimizer 為 AdamW,lr $= 10^{-4}$,weight decay $= 0.05$,$\beta_1 = 0.9$、$\beta_2 = 0.95$;前 1,000 steps 線性 warmup 至 10×,之後 cosine decay 至 $10^{-6}$。Inner (TTT) optimizer 採 Muon,5 次 Newton-Schulz 迭代,lr $= 0.1$,訓練時每層只跑 1 個 optimizer step。TTT 內部 MLP 採 SwiGLU,輸入/輸出維度 1024(對齊 VGGT hidden state),hidden layer 為 4× 擴張;loss 為 dot-product loss $L_t(T_\theta(k_i), v_i) = T_\theta(k_i)^\top v_i$。在 V 上額外套用 3×3 ShortConv2D,且僅 image patch token 的 V 參與 ShortConv2D,camera 與 register token 直接 pass-through(Sec. 推論時 default 使用 2 個 TTT optimizer step(Fig. 3a, Fig. 5);pointmap 採 global pointmap head 直接輸出,而非由 depth + camera unprojection(因 camera pose 預測較不精確,Sec. 4.1, Sec. Experiments — Ablations & Assessment

Ablations

於 ScanNet++、224×224 解析度、2–24 views 的小規模設定下進行(Tab. 6):

  • Linearization 來源(Tab. 6 i):若 TTT-from-scratch,CD 從 softmax 的 0.061 退化到 0.262,NC 0.844 → 0.727,mAA(30) 76.33 → 52.95。診斷:確認「先用 softmax 預訓再線性化」是必要 recipe 而非裝飾,屬有意義的 ablation。
  • 替代 linearization 方法(Tab. 6 ii–iii):T2R CD 0.137 / mAA 66.27、LoLCats CD 0.097 / mAA 62.87,均大幅落後本方法 (iv) CD 0.074 / mAA 72.16。診斷:在相同 base architecture 下比較,合理回答「為何選 TTT 形式的 linearization」。
  • ShortConv2D(Tab. 6 v / Tab. 9):加上 3×3 後 CD 0.074 → 0.066、NC 0.833 → 0.838、mAA 72.16 → 74.14;Tab. 9 進一步比較 V-3 / V-5 / K-3 / KV-3,顯示 5×5 沒有更好(0.069),只套在 K 微弱(0.068),同時套用在 K 與 V 反而退化(KV-3 CD 0.081),呼應 Sec. 3.2 中「同時做 spatial mixing 會破壞 K→V 解耦」的論述。診斷:屬有目的的設計驗證,直接連結到 §3.2 的可表達性論點。
  • TTT optimizer steps(Fig. 3 / Fig. 5):in-distribution(20 imgs)1 step 即足夠;1k–2k imgs 升到 2 steps 將誤差由 ~5× 退化壓回近常數,3、4 steps 並未進一步改善。診斷:回答 sequence-length generalization 的具體機制,屬診斷性實驗。
  • VGGT baseline 強化(Tab. 8):為了「公平比較」,作者對 VGGT 加上 entropy-scaling(Eq. (7))與 memory 優化,例如 1000 imgs 時 CD 0.041 → 0.029。診斷:此並非對本方法的 ablation,而是對 baseline 的增強,屬合理的對照處理但接近 sanity check。

整體 ablation 主要回答「pre-training + linearization 是否必要」、「為何不選 T2R/LoLCats」、「ShortConv2D 該套在哪」、「inference step 數如何隨 scale 調整」四個問題,屬診斷性而非單純 sanity check;唯獨缺乏對 Muon vs. 其他 inner optimizer、TTT MLP 寬度/深度的消融,這部分 the paper does not specify。

Weaknesses

4.2.1 Author-acknowledged

  • 相機姿態估計顯著弱於 $O(n^2)$ baseline:作者推測與 VGGT 的「dedicated camera token + image tokens」異質結構難以被 TTT 層中單一 MLP 同時記憶有關(§4.1、Tab. 3)。
  • 相對 softmax attention 仍存在精度差距,尤其在 wide-baseline 場景下,作者明列為主要 future work(§5 Limitations,page 8)。
  • 在空間範圍較大的 Waymo 序列上,部分複雜佈局仍出現重建退化(Fig. 9b、§D)。
  • Visual localization 精度與專用方法仍有顯著差距(Reloc3R 在 7Scenes 達 $e_r = 1.02^\circ$,本文為 $6.71^\circ$),作者明說只是 proof-of-concept(§4.3,page 8)。
  • 為了能在 query 階段預測相機,在視覺定位設定中仍保留所有 mapping 影像的 camera tokens 以餵入 VGGT 的 camera head softmax attention,亦即整體推論並非完全擺脫 softmax(Appendix A inference details,page 11)。

4.2.2 Phyra-inferred

  • VGGT baseline 在大規模實驗中被作者「悄悄加強」:Appendix B 引入 attention entropy-scaling 與 context-parallel 改寫,Tab. 8 顯示這些修改本身就使 VGGT 在 1000 影像時 CD 從 0.041 降到 0.029,因此 Tab. 1–4 中的「VGGT」並非公開 checkpoint,讀者難以據此校準與其他文獻的差距。
  • TTT3R 比較缺乏訓練資料與計算量對齊:作者僅將其視為 concurrent linear baseline,未控制 dataset、token 數、訓練步數,因此 Tab. 1/2/5 中對 TTT3R 的大幅領先有可能部分來自訓練資源而非架構優勢。
  • 大規模 pointmap 評估用 10 個 equally spaced keyframes 跑 ICP 對齊(Appendix A),這對輸出較平滑、低頻誤差較小的方法更有利,VGG-T3 的 MLP 隱式狀態本質偏向低頻,此評估設定可能放大其相對優勢。
  • Test-time scaling 結論建立在 1 vs 2 步的窄樣本上(Fig. 3a),Fig. 5 雖延伸到 4 步,但僅在 NRGBD 一個資料集驗證;若場景幾何遠比訓練分佈複雜,2 步固定設定能否維持「近乎常數」誤差缺乏壓力測試。
  • ShortConv2D 將 mAA 由 72.16 拉到 74.14,但與 softmax 上限 76.33 間仍有 2.2pp 缺口(Tab. 6),表示「打破 $K \to V$ 平凡解」只解掉一部分容量問題,文章把它定位為核心貢獻之一,實際影響相對偏小。
  • 「unordered 輸入支援」的優勢主要來自 TTT3R 在 unordered 上崩壞(Tab. 3 RPEr 從 0.617 暴升到 3.942);這比較等於拿一個 auto-regressive 模型在違反其輸入假設下對打,並不能直接證明 VGG-T3 對 unordered 場景具備正面優勢。
  • 固定大小 MLP 作為「整個場景的隱式狀態」缺乏容量隨場景複雜度成長的機制:Fig. 9b 已可見大空間範圍 Waymo 失效,但文章未量化「scene complexity vs MLP hidden dim」的 Pareto 曲線,讓 scaling claim 停在序列長度而非場景幾何複雜度。

Phyra's Judgment

7. Phyra's Judgment

7.1 Claimed vs. Supported Contributions

  • Claim (i): offline feed-forward 3D reconstruction 隨輸入視圖數線性 scale。Supported. Fig. 1b、Fig. 4、Tab. 4 在 100/500/1000/1500/2000 影像下皆證實 runtime 隨 $n$ 近乎線性,且 1k 影像 58 秒、2k 影像 48.5 秒(4 GPU)的端到端結果可重現此 claim。
  • Claim (ii): 變動長度 KV 隱式狀態可被「轉換」為固定維度隱式狀態而保留 global 聚合能力。Partially supported. Tab. 1/2 在 pointmap、video depth 上接近 $O(n^2)$ baseline,但 Tab. 3 camera pose 與 §5 wide-baseline 限制顯示「保留全域聚合」並不完全成立,固定 MLP 仍有結構性容量損失。
  • Claim (iii): 同時支援單卡大規模推論與分散式推論。Supported. Tab. 4 顯示 1 GPU/2 GPU/4 GPU 的線性加速,單卡在 1500 影像 173.1s、2000 影像 230.7s,VGGT 同條件 OOM。
  • Claim (iv): 統一 mapping 與 visual localization 於同一 feed-forward 模型。Partially supported / proof-of-concept. Tab. 5 顯示對 TTT3R 有清晰優勢,但相對 Reloc3R(7Scenes $e_r = 1.02^\circ$ vs 本文 $6.71^\circ$)精度仍遠遠落後;作者亦只把它定位為 feasibility demonstration 而非 SoTA localizer。
  • Implicit claim: TTT-based linearization 顯著優於既有 transformer→linear-attention 蒸餾方法。Supported within the chosen baselines. Tab. 6 顯示 mAA(30) 由 T2R 66.27 / LoLCats 62.87 拉到 74.14,惟對齊資源與資料的條件未控制,因此這只是「在本文設定下優於」而非全面性結論。

7.2 Fundamental Limitations of the Method

MLP 隱式狀態的容量瓶頸。 每層只用一個固定大小 SwiGLU MLP(input/output 1024,hidden $4\times$)壓縮整個場景的 KV;這個容量與場景幾何複雜度脫鉤。當輸入是大空間範圍、多分區、結構複雜的戶外駕駛場景(Fig. 9b)時,MLP 缺乏可隨 $n$ 或場景熵成長的自由度,品質退化在所難免。增加 TTT 步數只是在同一容量上做更精細的擬合,並無法繞過這個 information-theoretic 上限。

$K, V$ 同源線性投影的內在平凡解。 Eq. (1) 中 $K = W_k x, V = W_v x$,使 TTT 目標 $\min_\theta \sum_i \|T_\theta(k_i) - v_i\|$ 存在解析平凡解 $V = W_v W_k^{-1} K$。本文的 ShortConv2D 是一個 band-aid:讓 $V'$ 注入 2D 局部空間混合,迫使 MLP 學一個非平凡映射。但 Tab. 6 顯示其只把 mAA 從 72.16 拉到 74.14,與 softmax 上限 76.33 仍有 2.2pp 缺口,代表「破除平凡解」這一步並未真正補滿 softmax 的對比式檢索能力。任何把 $K, V$ 都從同一 token 線性投影出來的 TTT 變體都會繼承這個結構問題。

Camera pose 與 token 異質性的根本不相容。 TTT 層假設輸入 token 服從同一個 $K \to V$ 對應;但 VGGT 在每層 attention 之前插入專屬 camera token,其分佈與 image tokens 顯著不同。MLP 同時記憶兩種 modality 的 K→V 映射效率低落,Tab. 3 中 ATE/RPE 指標的全面退化即是此問題的外顯。要真正修補,需要對 camera token 設計獨立的 fast-weight 通道或保留 softmax 路徑,這已不是 hyperparameter 微調可解。

Wide-baseline 下的全域對比資訊被壓掉。 softmax attention 隱含跨 token 的 pairwise 對比與排序;TTT MLP 蒸餾後的 $T_\theta$ 只是個 amortized regressor,丟失了「在大量候選中找出真正匹配 view」這種長尾、稀疏匹配能力。作者在 §5 自承在 wide-baseline 上有 gap,但這不是 future work 的 hyperparameter 問題,而是 fixed regressor 取代 retrieval 的本質代價。

8.2 Improvement Directions

  1. (高可行)為 camera token 設計獨立 fast-weight 通道或保留 softmax 路徑。 作者已在 §4.1 自指 camera token heterogeneity 是 pose 弱化主因;只需在 global attention 層中對 camera token 走 softmax-over-camera-tokens、image token 走 TTT,即可保留 image side 的 $O(n)$ 同時恢復 pose 精度。對應 Tab. 3 中明顯落後的 ATE/RPE 指標,這是邊際收益最高的修補。
  2. (中可行)把最後 1–2 層 global attention 留給 softmax,前段全部 TTT 化。 wide-baseline 與精細匹配通常仰賴最後幾層的長距對比;混合架構可保住 §5 自承的 wide-baseline 缺口,且因為最後層 token 數已小(image tokens),softmax 成本可控,整體仍接近 $O(n)$。
  3. (中可行)以可學的 step-count predictor 取代固定 2 步 TTT。 Fig. 3a 顯示最優步數隨輸入規模分佈不同,固定 2 步是 heuristic;可由場景大小、token 多樣性等 cheap statistics 預測 optimizer steps,在小場景省算力、大場景加足夠步數,是 Fig. 5 直接暗示的 low-hanging fruit。
  4. (中可行)讓 MLP 容量隨輸入規模自適應(stacked / mixture-of-MLPs)。 Fig. 9b 的 Waymo 失敗暴露固定 MLP 的容量上限;一個簡單方案是 mixture-of-fast-weights,依據輸入 token 統計選擇/融合多個 MLP 專家,維持 mini-batch 梯度可分解性,因此仍能 CPU offload + DDP。
  5. (低可行)以 contrastive auxiliary loss 訓練 fast weights,補回 softmax 的 retrieval 能力。 softmax 的本質是 normalized retrieval;若在 TTT 內部加入「對其他 mini-batch 的 negative samples 做對比」的輔助目標,有望縮小 wide-baseline 差距。代價是訓練更複雜、需重新驗證 §3.3 的梯度可分解性,故排在後位。

Conclusion

(160 words)

§5 Conclusion 篇幅極短但定位清晰，純粹收束全文 thesis：作者重申 contribution 為「scalable feed-forward 3D reconstruction that gracefully scales with the number of input views」，核心機制是用 test-time optimization 學 $K \to V$ mapping 取代 softmax attention 的 KV querying。再以一組量化 punchline 收尾——VGG-T3 在 1k images 上 11.6× 加速、2k images 上達 33× 加速，同時在 pointmap 與 video depth estimation 上大幅優於其他 linear-time 方法。

Limitations 段落作者誠實坦承 trade-off：linear-time MLP scene representation 雖然成功保留 scalability 與 global aggregation 屬性，並大勝 auto-regressive 的 online 方法，但相對於 softmax attention 仍存在精度差距，特別是在 wide-baseline 場景。作者把這個現象 frame 成一個 open research question：如何 reconcile fixed-expressivity MLP scene representation 與 quadratic attention 的高精度。

雖然論文未獨立列出 Future Work 段，但結尾一句明確指出後續方向——尋找介於 fixed-size MLP 與 full softmax attention 之間、能 match softmax 精度卻保持 linear scaling 的更具表達能力 representation。Appendix D 補充的 Waymo wide-baseline 失敗案例，也呼應這個 future direction，並暗示自適應依場景複雜度調整 computation 的設計可能是一條路徑。
