2. Research Overview

2.1 Research Topic

本文聚焦於長序列、大尺度場景下的 feedforward 密集 3D 重建。現有 geometric foundation models（如 DUSt3R、VGGT、π3）雖能在短窗格內輸出高品質 pointmap，但受限於 bidirectional attention 的二次方複雜度，以及訓練資料多為數十至上百 frame 的「短窗泡泡」，難以擴展至數千乃至上萬 frame 的影片。作者提出 LoGeR，採用 chunk-wise 的因果處理流程，在每個 chunk 內保留強大的雙向幾何推理，並透過一個學習式的 hybrid memory 模組同時維持鄰接 chunk 的高保真對齊與長距離全域一致性。為了驗證在前所未有規模下的表現，作者重新利用 VBR 資料集，建立涵蓋最長 19k frame、軌跡達 11.5 km 的長序列重建基準，並在 KITTI、ScanNet、TUM、7scenes 等標準資料集上評估 ATE 與重建品質。

2.3 Core Architectures Used

  • Chunk-wise causal processing pipeline：將輸入影片切成具最小重疊（單一 overlapping frame）的多個 chunk，於每個 chunk 內保留 bidirectional 推理、跨 chunk 改以線性複雜度記憶機制傳遞資訊，是本文主架構骨幹。
  • π3 bidirectional geometry backbone：作為 chunk 內的 dense pointmap 與 camera pose 預測主幹，LoGeR 直接繼承其 patchifier、frame attention、chunk-wise bidirectional attention 模組與 prediction heads。
  • Sliding Window Attention (SWA)：作為非參數化的「短距無損資訊高速公路」，僅在 4 層網路深度插入，對前一個 chunk $C^{m-1}$ 與當前 chunk $C^m$ 的 frame attention 輸出做 cross-chunk 注意力，以保留高精度鄰接幾何細節。
  • Chunk-wise Test-Time Training (LaCT)：作為參數化的長距全域記憶，以 fast weights $W$（用 SwiGLU MLP 實作、Muon 優化器更新）逐 chunk 進行 apply-then-update，錨定全域座標並抑制 scale drift。
  • Hybrid memory block：每個殘差 block 依序串接 frame attention、SWA、TTT apply/update、chunk-wise bidirectional attention，以結構化方式整合三種時間尺度的幾何一致性。
  • **Feedforward SE(3) pose alignment (LoGeR\*)**：以重疊 frame 估算剛性對齊 $A^m = \tilde{T}^{(m-1)}_k (\hat{T}^{(m)}_k)^{-1}$ 並套用至整個 chunk，作為可選的 feedforward 校正步驟，於訓練與推論時皆可使用。
  • Pointmap & camera-pose decoders：沿用 π3 的輕量 decoder，輸出每像素 local pointmap 與 per-frame camera pose，並以 scale-invariant local loss、affine-invariant pose loss 與 global pointmap loss 共同訓練。

2.4 Core Argument

作者主張：feedforward 幾何重建之所以無法擴展到分鐘級影片，根因有二，一是架構上的「context wall」——bidirectional global attention 為了學到複雜幾何先驗而必須付出 $O(N^2)$ 代價，無法直接套用於長序列；二是訓練面的「data wall」——現有資料集集中於短窗、室內或小尺度場景，模型即使透過推論期最佳化（如 FastVGGT）擠出記憶體，也仍會在大尺度 VBR 上崩潰。要同時跨越這兩道牆，必須將序列拆成 chunk 以維持局部推理在訓練分佈內，再以線性複雜度的記憶機制處理跨 chunk 一致性。然而單一記憶機制都不夠：純 RNN/TTT（如 CUT3R）將所有時序壓進單一 hidden state，在鄰接 chunk 間損失高精度幾何細節導致接縫錯位；單純 sliding window attention 雖能無損保留近距資訊，卻沒有長距記憶以抑制 scale drift；naive deterministic stitching 則完全沒有全域記憶。因此作者提出「dual-component hybrid memory」——以非參數的 SWA 提供短距、無損的高精度對齊「資訊高速公路」，以參數化的 chunk-wise TTT（採 LaCT）作為固定大小、線性複雜度的長距全域記憶來錨定全域座標、抑制尺度漂移；兩者作用於不同時間尺度而互補，且能用 progressive curriculum 強迫模型把全域責任轉移至 TTT。再加上以 TartanAirV2 等大尺度資料為主的訓練混合與可選的 feedforward pose alignment，模型只需在 128 frame 上訓練即可推論至 19k frame，在 KITTI ATE 上比前一代 feedforward 方法降低逾 74%。

3.1 Title and Abstract

(180 words)

論文標題 "LoGeR: Long-Context Geometric Reconstruction with Hybrid Memory" 直接點出三個核心關鍵字：long-context（長序列）、geometric reconstruction（幾何重建）、hybrid memory（混合記憶）。標題即承諾要把 feedforward 3D 重建從短窗口推到 minutes-long 影片尺度，且解法是「記憶混合」而非單一記憶機制。

Abstract 以「問題－方法－結果」三段式佈局展開故事。問題端先陳述現況：feedforward geometric foundation models 在短窗口 (short-window) 表現強，但放大到 minutes-long videos 時受限於 quadratic attention complexity 或 recurrent design 的有限有效記憶。方法端提出 LoGeR：以 chunk-wise 處理影片串流，chunk 內保留 bidirectional priors 的高保真推理，chunk 間以 learning-based hybrid memory module 維持一致性，由兩個互補元件組成：parametric Test-Time Training (TTT) 錨定 global coordinate frame、防止 scale drift；non-parametric Sliding Window Attention (SWA) 保留未壓縮的鄰近上下文以利高精度對齊。

結果端承諾兩件量化事實：在 128 frames 訓練可推廣到推論時 thousands of frames；在 KITTI 上 ATE 相對 prior feedforward state-of-the-art 降低逾 74%，並在重新整理的 VBR 資料集（最長 19k frames）上達到 robust globally consistent 的結果。Abstract 為後續 §1 introduction 的「context wall + data wall」雙牆論述鋪陳了先決條件。

3.2 Introduction

(720 words)

Introduction 採取「典範轉移－雙重瓶頸－設計取捨－混合記憶提案－基準與成效」五拍敘事。第一拍鋪陳大尺度 dense 3D reconstruction 在 holistic scene understanding 與 generative world-building 的核心地位，並對比 classical optimization frameworks 與 geometric foundation models（DUSt3R、MonST3R、VGGT、π3）的兩種典範：前者可重建整座城市但需離線且在 sparse / texture-less 輸入失靈；後者具強 feedforward 推論能力卻 confined to bounded scenes，從而界定本文要填補的空缺。

第二拍將阻礙形式化為兩堵牆。Architectural 的 "context wall"：bidirectional attention 對學習複雜幾何先驗不可或缺，但其 quadratic complexity 限制其只能用於短上下文窗口。Data 的 "data wall"：現行模型訓練資料以 dozens to over a hundred frames 的「短上下文 bubbles」為主，缺乏整合 thousands to tens of thousands of frames 長距依賴的能力。FastVGGT 即便在記憶體上鬆綁，仍在 VBR 上失敗（Figs. 1 and 3），指認 inference-time heuristics 解不開 data wall。

第三拍提出 end-to-end chunk-wise processing 為實用策略：分段確保 local inference 落在 in-distribution 範圍，並能直接複用 π3 等強 bidirectional baseline。然而此典範引入新問題——chunk 邊界的記憶與一致性管理。作者把 dense 3D reconstruction 所需的 coherence 拆成三層：intra-window details、uncompressed context for high-precision local alignment、global structural integrity。並指出既有 sequential models 無法同時兼顧：CUT3R 將時間 context 壓進單一 lossy hidden state，犧牲鄰近對齊所需的 dense 高精度資訊；naïve deterministic stitching 雖保留局部細節但缺乏防止 scale drift 的長距記憶。

第四拍引出本文核心立論：單一 memory strategy 在本質上不足，必須走 hybrid memory module。Table 1 的對照表將 Full Attention、SWA、TTT/Linear Attention、Ours (Hybrid Memory) 在 compute cost、local context、global context 三軸並列，將「Lossless local + Compressed global + O(N)」定位為 LoGeR 的設計座標。具體做法是把 non-parametric SWA 用於 most recent chunks 的高保真鄰接傳遞、把 parametric TTT 用於壓縮全域上下文以錨定座標系，並明確主張兩者「decouple」——長程 parametric memory 防 scale drift、短程 non-parametric memory 保 high-precision transitions。

第五拍同時鋪陳 benchmark 與成效。為了在前所未見的尺度上評估，作者基於 VBR 資料集打造長序列重建 benchmark，最高達 19k frames、11.5 km 軌跡。架構設計使模型能在 128 frames 訓練、推論時 generalize 到 1k frames，並在 periodic state resets 與 optional feedforward pose-alignment 加持下擴張到 19k frames。引言以兩個量化結果作為 §5 的下游 anchor：KITTI ATE 從 72.86 降到 18.65、VBR 上 55.2% 相對提升，把 §3 preliminary 與 §4 approach 的閱讀預期錨定在「兩堵牆同時被 hybrid memory 攻破」這條主線上。

3.3 Related Work / Preliminaries

(900 words)

§2 Related Work 與 §3 Preliminary 共同承擔三項功能：定位本文於既有 visual SLAM 與 feedforward 3D reconstruction 兩條學術脈絡之中、把長序列重建現況收斂為三條失敗模式、為 §4 的 hybrid memory 模組提供必要的 TTT 數學介面。

§2 先以四個子主題盤點先驗工作。Learning-based visual SLAM 將 DROID-SLAM、DPV-SLAM 與 VGGT-Long、VGGT-SLAM 並列為「以 backend 維持長距 coherence 的唯一可行路徑」，但同時點出其昂貴的 graph construction、loop closure、global optimization 後端，為本文「全 feedforward 也能贏」的論斷劃出對照組。Feedforward 3D reconstruction 一段把 DUSt3R、MASt3R、MonST3R、VGGT、π3 串成一條主線，並指出其共同弱點：受 global attention 的高記憶體成本所限、僅能處理 bounded scenes，從而無法滿足 robotics 與 autonomous driving 對 global consistency 的需求。Long sequence reconstruction 一段細分四種既有路線——external spatial memory（Spann3R、Long3R、Point3R、Stream3R）、RNN-style persistent state（CUT3R）、sparse / causal VGGT 變體（FastVGGT、InfiniteVGGT、Streaming 4D VGGT、LongStream），並把 TTT3R 的 frame-wise confidence-based update 點名為「缺乏多影格 reasoning expressivity」的代表，藉此把 LoGeR 的 chunk-wise + bidirectional + hybrid memory 與其區隔。

第四個子主題 Memory for long-context modeling 把視野拉到 NLP 與通用 sequence modeling：state-space models（S4、Mamba）、linear attention（Katharopoulos et al.、Schlag et al.）、local attention（Longformer、BigBird）與 fast-weight 的 TTT。Table 1 的取捨被在此處正式化——個別機制都在 compute、local、global 三軸間有不可避免的折衷。作者也明確承認，語言模型常見的 hybrid 配方（Longformer、Jamba）會搭配 full global attention，但 dense vision 的 token density 讓此路徑在計算上不可行，因此 LoGeR 才必須採用嚴格 linear-in-N 的 SWA + TTT 組合。Dataset 段落則為 §4.3 的 "data wall" 預先列出可用訓練資源（ARKitScenes、DL3DV、MegaDepth、ScanNet、Waymo、HyperSim、Spring、TartanAir/V2、UnReal4K、Virtual KITTI 2、OmniWorld-Game），並指出既有 benchmark 的空間範疇限制，因而需採用 VBR 作為 8,815–18,846 frames、11.5 km 等級的長序列尺規。

§3 Preliminary 為後續方法章節提供最小必要符號。第一段定義 3D reconstruction model 的輸入/輸出介面：給定 $N$ 張 multi-view 影像 $I = \{I_i\}_{i=1}^{N}$，每張 $I_i \in \mathbb{R}^{H \times W \times 3}$，bidirectional transformer 預測局部 pointmap $P_i \in \mathbb{R}^{H \times W \times 3}$ 與 camera pose $c_i \in \mathbb{R}^{4 \times 4}$。第二段引入 Test-Time Training：把 fast weights $W$ 與 slow weights 區分，對於 token 序列 $x = [x_1, \dots, x_N]$, $x_i \in \mathbb{R}^d$，標準 attention 投影為 $q_i, k_i, v_i$，TTT 把 $f_W(\cdot): \mathbb{R}^d \to \mathbb{R}^d$ 視為 fast-weight 參數化的神經網路，並列出兩個操作：

$W \leftarrow W - \eta \nabla_W L\bigl(f_W(k), v\bigr)$

$o = f_W(q)$

並補充實作細節：fast weight 採 SwiGLU layer、優化器採 Muon，並承認 TTT 對長 context 的線性複雜度伴隨 lossy 壓縮。此節因此清楚把「TTT 提供 compressed global memory 但有 inherent lossiness」植入讀者腦中，為 §4 中以 SWA 補足 lossless local context 的必要性鋪軌。

3.4 Method (overview narrative)

(1300 words)

§4 Approach 整章的敘事弧為：先以「為什麼是 chunk-wise」為前提，再從 chunk 邊界 coherence 拆出三項需求，導出 SWA + TTT 的 hybrid memory 組合；接著以單一 block 的四步驟流水線把架構落地；最後提出三項損失與 LoGeR* feedforward alignment 的可選變體，並以 data 與 curriculum 策略補上資料側的設計。

§4.1 Long-Context 3D Reconstruction Architecture 從 motivation 開場：要把 feedforward dense 3D reconstruction 推到 minutes-long videos，必須同時克服 global attention 的 quadratic complexity 與 long-horizon training data 的稀缺，end-to-end chunk-wise processing 因此是自然解。但獨立處理 chunk 會破壞 global consistency，故架構必須同時提供三件事：(i) chunk 內強 bidirectional reasoning 以保 dense geometric fidelity；(ii) chunk 邊界 lossless 短程資訊高速公路 以保高精度對齊；(iii) 線性時間、固定大小的長距 memory 以跨越 thousands of frames。Overview 段把輸入串流 $X = \{I_t\}_{t=1}^{T}$ 切成 $M$ 個 chunk $\{C_m\}_{m=1}^{M}$，採最小 overlap（如單一 overlapping frame），chunk 內部以 VGGT 或 π3 等 bidirectional backbone 處理，跨 chunk 則同時引入兩條互補通路。

第一條為 long-term lossy 壓縮的 chunk-wise TTT：每個 residual block 中插入 TTT layer，維持一組跨多 chunk 的 fast weights $W$，並採用 Large-Chunk TTT (LaCT) 以對齊 chunk-wise 處理；apply 操作把歷史資訊送進當前 chunk、update 操作把當前 chunk 的 coarse geometry 與 scale 等冗餘但重要資訊壓回 $W$。雖然 fast weights 理論上具無限 receptive field，但實務 capacity 受訓練 context length 限制，因此推論時可選擇性地以 periodic state resets 防止超長 horizon 的飄移。第二條為 short-term lossless 傳遞的 SWA：作者觀察到 TTT-only state passing 對 dense 3D 中鄰近影格的幾何一致性而言是有問題的，因此 sparsely 插入 SWA layer，使其同時 attend 前一個與當前 chunk 的 frame attention 輸出 $C_{m-1} \cup C_m$，建立一條把高保真特徵直接從前 chunk 推進當前 chunk 的 lossless 高速公路。重要的是，SWA 僅在相鄰 chunk 間運算且只放在四層深度，因此 compute 與 memory 維持在 bounded 範圍。兩條路徑互補：TTT 提供 scalable long-range memory、SWA 確保 fine-grained 鄰接幾何一致性。

Block structure 段落把上述設計拆成單一 block 內的四步流水線，並標出對應方程式。slow weights $\theta$ 在推論時凍結；對 chunk $C_m$ 的 token 序列 $H_{C_m}$ 依序執行：(1) per-frame attention 以提取空間特徵 (Eq. 3)；(2) sparse SWA 跨 $C_{m-1}$ 與 $C_m$ 對齊鄰近 chunk (Eq. 4)；(3) chunk-wise TTT layer 以 apply-then-update 程序整合全域 context，先把當前記憶注入 token、再把當前 chunk 摘要寫入 $W$ (Eqs. 5, 6)，並用 pre-norm 穩定 long-horizon streaming；(4) chunk-wise bidirectional attention 在 chunk 內做強幾何推理 (Eq. 7)。block 末端的 prediction heads 沿用 π3 的 lightweight pointmap decoder 與 camera-pose decoder，輸出 dense pointmap 與 per-frame camera pose。

§4.2 Learning Objectives 接續 π3 的訓練哲學，採三項 reference-view-free 損失：scale-invariant local pointmap loss $L_{local}$（用單一 per-sequence scale $s^$ 對齊預測，並以深度 $z_{i,p}$ 正規化）、affine-invariant relative pose loss $L_{pose}$（rotation 用 $L_{rot}$、translation 用 Huber 損失並乘上 $s^$）、與額外的 global pointmap loss $L_{global}$（在 world-coordinate pointmap 上監督，以 over-constrain 長序列訓練）。總損失為 $L = L_{local} + L_{pose} + \lambda_{global} L_{global}$，符號 $\Omega$、$\hat{x}_{i,p}$、$\hat{T}_i = [\hat{R}_i \mid \hat{t}_i]$、$\Pi(T, x)$ 等與標準 pointmap 訓練介面對齊。

LoGeR* 是一個重要的 feedforward alignment 變體：在很長的串流中，預測誤差仍可能累積，因此於 chunk 邊界以 overlapping frame 估出剛體對齊 $A_m = \tilde{T}^{(m-1)}_k \bigl(\hat{T}^{(m)}_k\bigr)^{-1}$，並把當前 chunk 的所有 pose 變換為 $\tilde{T}^{(m)}_t = A_m \hat{T}^{(m)}_t$，作為訓練與推論時最終的 camera pose 預測。

§4.3 Data and Curriculum 把資料側的「data wall」明確化：純架構改動不足以達成無限上下文重建，VGGT + FastVGGT 在僅以短上下文資料訓練時無法泛化到大尺度場景（Fig. 3），因此必須以 large-scale-scene 資料集（特別是 TartanAirV2）為主軸建構訓練混合。Curriculum 採三階段：48 frames / 4 chunks → 12 chunks（固定序列長度）→ 在 H200 上把 context length 提升到 128 frames、最多 20 chunks，迫使模型逐步把依賴從 SWA 移交到 TTT hidden state；LoGeR* 則自第一階段模型起初始化、加入 feedforward alignment 後完成剩餘 curriculum，兼顧訓練效率與最終效能，並為 §5.3 ablation 預留可比較的設計變因。

3.5 Experiments (overview narrative)

(1100 words)

§5 Experiments 以「實作設定 → 長序列評估 → 短序列評估 → ablation」四層展開，整體論證策略是先在最具挑戰的 long-horizon 設定上拉開差距，再回頭證明 LoGeR 在短序列上仍是 state-of-the-art，最後以 ablation 把每一個架構與資料決策的貢獻釘住。

實作細節段落首先列出訓練資料（ARKitScenes、DL3DV、HyperSim、MegaDepth、ScanNet、ScanNet++、Spring、TartanAir、TartanAirV2、UnReal4K、Virtual KITTI 2、Waymo、OmniWorld-Game 子集），交代 AdamW、40k steps、batch size 32、32 張 H100 兩天 + 32 張 H200 兩天的訓練配置；patchifier、frame attention 與 chunk-wise bidirectional attention 由 π3 初始化；所有評估在單張 NVIDIA A100 40GB 上完成。這段把後續所有結果置於同一 hardware budget 之下，使表格之間可比。

§5.1 Evaluation on Long Sequences 是全章重心。Benchmark 端選用 KITTI（最多 4,661 frames、5 km 軌跡）與重新整理的 VBR（7 條 8,815–18,846 frames、1.4–11.5 km 軌跡）。誤差指標統一採 Umeyama-aligned 後的 ATE，符合 Teed & Deng、Lipson et al. 的標準。Baselines 同時涵蓋 optimization-based（DROID-SLAM、DPV-SLAM、VGGT-SLAM、VGGT-Long）與 feedforward（CUT3R、TTT3R、FastVGGT、InfiniteVGGT），並額外以 π3 衍生出 Pi3-Chunk（chunk-wise 處理 + SIM(3) 對齊）作為新 baseline；CUT3R 與 TTT3R 採 TTT3R 提出的 reset 算法以取得合理結果，LoGeR 則每五個 window 重置一次 fast weights，重置時亦套用 feedforward pose alignment。

結果上，LoGeR 與 Pi3-Chunk 在 KITTI 上同時超越所有既有 feedforward 方法（Tab. 2），LoGeR* 平均 ATE 從 TTT3R 的 72.86 降到 18.65；LoGeR 的平均 ATE 25.44 甚至比最強 optimization-based 方法 VGGT-Long 高出 32.5%，且優勢主要來自 open-loop 軌跡（01、03、04、08、10），顯示其在無 loop closure 的條件下也能抑制累積飄移。VBR 端的故事更銳利：Pi3-Chunk 在約 1k frames 時略優，但其 SIM(3) 對齊依賴 local overlapping frames 估計 scale，誤差會在長距下指數累積；LoGeR 的 TTT 模組從架構上錨定 global scale，於 20k frames 仍維持 global scale consistency，最終取得 55.2% 相對改進（Fig. 4），質性軌跡圖（Fig. 5）也佐證 baseline 出現嚴重 scale drift 而 LoGeR 緊貼 ground truth。

§5.2 Evaluation on Short Sequences 把評估擴充到最多 1k frames：7-Scenes 上的 3D point-cloud reconstruction（Figs. 6, 7）、ScanNetV2 與 TUM-Dynamics 上的 camera pose estimation（Figs. 8, 9），baseline 涵蓋 explicit-state（Point3R）、implicit state-space（CUT3R、TTT3R、StreamVGGT）與 bidirectional（VGGT、π3）。所有實驗統一採 chunk size 64、overlap 3。Pi3-Chunk 與 LoGeR 在所有對比上皆顯著勝出，並回應了「chunk-wise + bidirectional 不只贏在長序列」這條敘事線；雖然 Pi3-Chunk 在小尺度 TUM 上 pose 指標稍佔上風，但 LoGeR 重建出的 bookshelf、室內結構在視覺上明顯較少幾何扭曲。

§5.3 Ablation 把架構、資料、curriculum 三項決策分別釘下。Architecture 端，分別 inference-time 關閉 SWA 與 TTT，並在 ScanNet 子集與 TUM-Dynamics 上比較 ATE（Tab. 3）：兩者皆下降，且 Fig. 10 的軌跡圖把分工釘得很清楚——TTT 維持 global consistency、SWA 維持 local smoothness，缺一即出現顯著飄移或幾何退化。Dataset 端，移除 TartanAir、TartanAirV2、Waymo、Virtual KITTI 2、OmniWorld-Game 五個 large-scale 資料集後性能顯著下降，驗證「data wall 必須以 large-scale navigation 資料破解」的假說。Curriculum 端，與不採 curriculum 的訓練 schedule 對照下，progressive 策略對 LoGeR 與 LoGeR* 兩者皆帶來一致改進，回應 §4.3 把 reliance 從 SWA 移交到 TTT hidden state 的訓練設計初衷。整章 §5 因此構成一條完整論證鏈：方法在 long-context 開出最大差距、在 short-context 仍是 SOTA、所有設計選擇都可由 ablation 個別解釋。

3.6 Conclusion / Limitations / Future Work

(290 words)

§6 Conclusion 與其後的 Discussion and Future Work 段落把整篇論文收束於三條可延伸軸線。Conclusion 重申本文核心貢獻：以 SWA 與 TTT 互補的 hybrid 架構解決 feedforward 3D geometry 模型在長序列估測上的限制；在所提 VBR benchmark 上，LoGeR 不仰賴 optimization 即可處理 thousands of frames，並在多種記憶表徵（explicit memories、recurrent states、causal attention）的對手中取得最佳長序列重建結果。作者把該成果定位為 dynamic scenes、video understanding 與 robotics 等 long-context spatio-temporal reasoning 任務的新起點。

Discussion and Future Work 則誠實點名兩項開放問題並暗示一條延伸方向。第一項是 TTT fast weights 的長度泛化：雖然其固定大小理論上允許無限 context，但實務上難以泛化到超出訓練 chunk 數的長度（引 Ruiz & Gu, 2025），實際有效範圍受 hardware memory 限制；超過此範圍會累積 trajectory drift，目前只能以 periodic state resets 緩解，但代價是犧牲長期 context。作者明確期待未來 linear sequence model 解決此 length-generalization 瓶頸。第二項是「data wall」：高品質、長 horizon、夠多樣的訓練資料仍是關鍵瓶頸，呼籲社群投入長序列資料的 curation。第三項則為延伸：把 SWA + TTT 的 hybrid memory 推到幾何重建以外、需要同時兼顧長距全域一致性與強局部依賴的領域。

整體而言，§6 把全文限制與 §4.3 的 data 哲學、§5.1 的 reset 策略與 §5.3 的 curriculum 結果連成一條可被後續工作延伸的軌道，避免把 LoGeR 推銷為終局解，而是把它定位為「在現行 hardware 與 dataset 限制下的可行混合策略」，並把長度泛化與資料策展兩個結構性問題交給社群。

4. Critical Profile

4.1 Highlights

  • LoGeR 在 KITTI 上把 feedforward 方法的平均 ATE 從前一代最佳 $72.86$ m（TTT3R）壓到 $18.65$ m，相對下降逾 $74\%$，並在 average 指標上勝過 optimization-based 的 VGGT-Long $32.5\%$（Tab. 2，p. 7）。
  • 模型只在 $48$/$128$ frame 上訓練，卻能直接 inference 到 $19{,}000$ frame、軌跡達 $11.5$ km 的 VBR 序列，且相對 prior feedforward 方法整體進步 $55.2\%$（Fig. 4，p. 7；Tab. 6，p. 15）。
  • 提出的 hybrid memory 在 Tab. 1 中明確界定設計位置：用 non-parametric SWA 維持 lossless 局部 context、用 parametric TTT 做 compressed 全域 context，整體仍維持 $O(N)$ 複雜度（p. 2）。
  • 7-Scenes 重建的 Chamfer Distance 比 prior work 改善 $69.2\%$（Fig. 6，p. 7），ScanNet 與 TUM-Dynamics 的 ATE 分別提升 $80.0\%$ 與 $66.1\%$（Fig. 9，p. 8）。
  • Bonn 上 $500$ frame 的 video depth Abs Rel 比 TTT3R 再降 $21.05\%$（Fig. 11，p. 15），顯示 hybrid memory 對局部 dense 預測同樣有正面效果。
  • Ablation 同時驗證 SWA 與 TTT 缺一不可：拿掉 TTT 在 ScanNet $1000$ frame 從 $0.107$ 漲到 $0.162$，拿掉 SWA 漲到 $0.143$（Tab. 3，p. 9）。
  • Curriculum training 與大尺度資料混合（TartanAirV2 等 5 個資料集）為 generalization 關鍵：移除大尺度資料後 ScanNet $1000$ frame 從 $0.107$ 退化至 $0.156$（Tab. 3，p. 9）。
  • LoGeR$^{*}$ 變體加入 feedforward SE(3) alignment，在 KITTI 上把平均 ATE 進一步從 $25.44$ 推到 $18.65$，顯示 chunk 對齊與 TTT memory 可以互補（Tab. 2，p. 7）。
  • 提出的 Pi3-Chunk 是一個簡潔但實用的 SIM(3) stitching baseline，本身就在 KITTI 上勝過 TTT3R 與 CUT3R（$52.07$ vs $72.86$ / $91.62$，Tab. 2，p. 7），凸顯 chunk-wise causal 處理本身的價值，再用 hybrid memory 進一步壓榨。
  • 提出 VBR 重新標註的 long-context 評測協議，序列長度 $8{,}815$–$18{,}846$ frame、軌跡 $1.4$–$11.5$ km，是目前 feedforward 文獻中最長的 dense 重建 benchmark（Sec. 5.1，p. 7）。

4.2 Weaknesses

4.2.1 Author-acknowledged

  • TTT fast weights 雖名義上具無限 receptive field，實際上難以 generalize 超過訓練時看過的 chunk 數，因此超過約 $1{,}000$ frame 後仍會累積 drift，必須靠 periodic state reset 補救，而 reset 本身就「sacrifice long-term context」（Discussion，p. 9）。
  • 大規模、長 horizon、且兼具品質與多樣性的訓練資料仍是瓶頸，作者明言「the availability of expansive, high-quality training data remains a significant bottleneck」並把 data wall 列為公開問題（Discussion，p. 9；Sec. 4.3，p. 6）。
  • 推論期效率仍有改進空間：作者承認可進一步 prune TTT 層或對 SWA 做 strided sampling，但「leave these system-level optimizations for future work」（Appendix A.4，p. 14）。
  • 在大尺度場景時，base LoGeR 比 LoGeR$^{*}$ 更易 drift，需手動把 KITTI 的 chunk size 從 $64$ 降到 $32$、VBR 降到 $48$ 才能拿到合理結果（Appendix A.4，p. 14），實質暴露 TTT memory 對 chunk size 的敏感性。

4.2.2 Phyra-inferred

  • 架構上沒有任何 loop closure 機制，因此在 KITTI 含 loop 的長序列（00、05、06）上仍輸給 optimization-based 方法：LoGeR$^{*}$ 在 00 是 $30.47$ vs DPV-SLAM++ 的 $8.30$、VGGT-Long 的 $8.67$；05 是 $26.34$ vs $5.74$ / $8.31$（Tab. 2，p. 7），而論文僅在「open-loop trajectories」上強調勝出，回避了這個結構限制。
  • LoGeR$^{*}$（即多加一個 feedforward SE(3) alignment）幾乎在所有 benchmark 都比 base LoGeR 好（KITTI $25.44 \to 18.65$、VBR $36.16 \to 31.75$、ScanNet/TUM 短序列也是，Tab. 2、Tab. 6、Tab. 3），這暗示 TTT 全域 memory 並沒有真正錨定座標，最後一公里仍是 deterministic alignment 在做，削弱了「parametric memory anchors the global coordinate frame」這項核心論述。
  • VBR 為單一城市（Rome）的 LiDAR-derived benchmark，所有 7 條序列同分布、同 sensor，論文用它代表「unprecedented horizons」但完全沒有跨城市或跨領域的 long-horizon 測試，generalization 未驗證（Sec. 5.1，p. 7；Tab. 6，p. 15）。
  • 「infinite context」與「trained on 128 frame, generalize to 19k frame」的敘事被自家 inference protocol 削弱：每 $5$ chunk 就要 reset TTT fast weights，且 reset 時又要靠 feedforward alignment 維持連貫（Sec. 5.1 Baselines，p. 7），這代表實際長距 memory 視窗大約只有 $5$ chunks，遠短於宣稱的 horizon。
  • 推論吞吐量在 A100 40GB、$500$ frame、chunk size $64$ 時僅 $9.3$ FPS（Tab. 5，p. 14），低於即時 SLAM，且論文未報告 $19$k frame 級別的 wall-clock，feedforward 對 SLAM 的「速度優勢」在此 regime 是否仍成立，未被檢驗。
  • 新引入的 TTT + SWA 帶來 $296$M 額外參數（佔 base $954$M 的 $\sim 31\%$，Appendix A.2，p. 13），但論文的效率比較全程只看 attention 複雜度與 OOM，沒有把這 $\sim 1.25$B 規模對 baseline 重新做 parameter-matched 對照，因此「hybrid memory 帶來的提升」與「單純 scale-up」混在一起難以分離。
  • ScanNet 與 TUM-Dynamics 在 $200$–$500$ frame 區段，Pi3-Chunk 與 LoGeR 表現相當接近（Fig. 9，p. 8），且 TUM 短序列上 Pi3-Chunk 的 pose 指標甚至略好（Fig. 8 caption，p. 8），這暗示在中短序列、小尺度場景下 hybrid memory 的邊際效益可能小於額外參數成本。
  • 關鍵 ablation 是在「縮短訓練 frame、子集評測」設定下做的（Sec. 5.3 與 Tab. 3，p. 9），SWA 與 TTT 的相對貢獻在最終 $128$ frame、完整資料、$19$k frame inference 的 regime 下並未重新驗證，無法保證 ablation 結論對最終 deployment 設定有效。

4.3 Phyra's Judgment (summary)

LoGeR 真正新穎的部分是把 SWA 與 chunk-wise TTT（LaCT）這個 language-model 領域熟知的 hybrid memory pattern 工程化到 dense 3D reconstruction 上，並給出第一個 minute-scale、kilometre-scale 的 feedforward benchmark 與相應的 curriculum、資料配方，工程貢獻紮實。然而它對「context wall + data wall」的雙重論述，主要是說明了「為什麼必須做 chunk-wise + 大尺度資料」，而 hybrid memory 本身對全域一致性的貢獻其實被 LoGeR$^{*}$ 的 feedforward SE(3) alignment 與 periodic reset 大量稀釋，TTT 是否真的「anchor global coordinate」仍是未解問題。整體仍屬於 architecture-level 的 incremental advance：在沒有 loop closure 的 feedforward 子群中是新的 SOTA，但相對 optimization-based SLAM 的核心結構性弱點（無 loop closure、長距 memory 實際視窗有限）並未解決。

5. Methodology Deep Dive

5.1 Method Overview

LoGeR 的核心動機是同時打破 feedforward 3D reconstruction 在長序列上面臨的兩堵牆: (i) 全域注意力的二次複雜度導致的 context wall,以及 (ii) 既有 short-context 訓練資料無法涵蓋分鐘級或公里級軌跡的 data wall (page 4, Sec. 4.1; page 6, Sec. 4.3)。為此,作者選擇 chunk-wise causal processing 作為基本範式: 將輸入視訊 $\mathcal{X}=\{I_t\}_{t=1}^T$ 切成 $M$ 個 chunks $\{\mathcal{C}^m\}_{m=1}^M$,僅以單一重疊 frame 連接相鄰 chunk (page 4),這既將每次 forward 的計算量限制在訓練資料分布之內,也保留了 bidirectional backbone (VGGT 或 $\pi^3$) 的強多 frame 推理能力。

跨 chunk 的資訊傳遞由 hybrid memory 雙路徑承擔 (Fig. 2, page 4)。第一條路徑是 Sliding Window Attention (SWA),僅在四個 layer 插入,讓當前 chunk $\mathcal{C}^m$ 的 tokens 直接 attend 到上一個 chunk $\mathcal{C}^{m-1}$ 的對應 layer 輸出 (Eq. 4),提供 lossless、bounded-cost 的鄰近幾何對齊。第二條路徑是 Chunk-wise Test-Time Training (TTT),維護一組 fast weights $W^m$ 沿 chunk 軸線性遞推,在每個 chunk 內先以 apply 操作將歷史記憶注入 token (Eq. 5),再以 update 將當前 chunk 摘要寫回權重以得 $W^{m+1}$ (Eq. 6)。fast-weight module $f_W$ 採用 SwiGLU MLP 並以 Muon optimizer 執行 test-time 更新 (page 4, Sec. 3 結尾),具體規格採用 LaCT (Large-Chunk TTT, Zhang et al., 2025b),比逐 token TTT 更具計算效率 (page 4)。Block 內部在 SWA 與 TTT 之後仍保留 chunk-wise bidirectional attention (Eq. 7),確保 chunk 內 dense geometric reasoning 的完整性。

訓練端用 progressive curriculum 三階段 (page 6, Sec. 4.3): (1) 48 frames / 4 chunks (12 frames/chunk) 的暖啟動;(2) 在固定 sequence length 下逐步加密到 12 chunks 以提升 chunk 邊界密度;(3) 切換到 H200 GPU,將 context 拉到 128 frames、漸進到 20 chunks。Loss 沿襲 $\pi^3$,組合 scale-invariant local pointmap loss $\mathcal{L}_{\mathrm{local}}$ (Eq. 8)、affine-invariant relative pose loss $\mathcal{L}_{\mathrm{pose}}$ (Eq. 9) 與 global pointmap loss $\mathcal{L}_{\mathrm{global}}$ (Eq. 10),總 loss 為 $\mathcal{L} = \mathcal{L}_{\mathrm{local}} + \mathcal{L}_{\mathrm{pose}} + \lambda_{\mathrm{global}} \mathcal{L}_{\mathrm{global}}$ (Eq. 11)。為了進一步壓制超長序列下的累積漂移,作者另外提出 LoGeR* 變體: 對每個 chunk 中的 overlapping frame $k$ 計算 SE(3) 剛體對齊 $A_m = \tilde{T}^{(m-1)}_k (\hat{T}^{(m)}_k)^{-1}$,並將 $A_m$ 套用到整個 $\mathcal{C}^m$ 的 raw poses (Eq. 12, page 5)。Inference 階段也可選擇每 5 個 window 對 fast weights 做 periodic state reset 以避免漂移 (page 4 結尾,page 7 Sec. 5.1)。

5.3 Per-Module Breakdown

Module 1: DINO Patchifier

Function: 將每個 frame $I_i \in \mathbb{R}^{H \times W \times 3}$ 切成不重疊 patches 並投影到 token embedding 空間,作為後續 hybrid memory blocks 的輸入。

Input: per-chunk frame tensor $\{I^m_j\}_{j=1..n}$,shape [B, M, n, H, W, 3] (page 3 Sec. 3 開頭定義 $I_i \in \mathbb{R}^{H \times W \times 3}$)。

Output: per-frame token grid $H^{\mathcal{C}^m}_j$,shape [B, M, n, HW/p^2, d]。

Processing: 對每個 frame 獨立執行 DINO 風格 patchification — 用 stride-$p$ kernel 將影像切成 $H/p \times W/p$ 個 patches,展平後線性投影到 $d$ 維 token embedding。chunk 內所有 frame 共享同一份 patchifier 權重 (page 5「first patchifies images into tokens」)。

Key Formulas: 該層為標準 patch embedding,paper does not specify the exact formula。

Implementation Details: paper does not specify 補丁大小 $p$、embedding 維度 $d$ 或所使用的 DINO 變體 (DINOv2 / DINOv3),因此 §5.2 中所有 token 維度上的 $p$、$d$ 皆為 ?。Curriculum 第一階段從 $\pi^3$ (Wang et al., 2026) 的 patchifier、frame attention 與 chunk-wise bidirectional attention 模組權重初始化 (page 6 Sec. 4.3 結尾)。

Module 2: Per-Frame Attention (Eq. 3)

Function: 在每個 frame 的 token 集合內部執行 self-attention,抽取 spatial 結構而不混入跨 frame 訊號。

Input: 單 frame token 張量 $H^{\mathcal{C}^m}_j$,shape [B, M, n, HW/p^2, d] (page 5)。

Output: 同 shape 的更新後 token 張量,殘差結構。

Processing: 對 $i \in \{1, \dots, n\}$ 各 frame 獨立呼叫 frame-level attention,使用 LayerNorm 預先正規化,再以加性殘差回寫。slow-weights $\theta$ 共享於所有 frame (page 5 (1))。

Key Formulas:

$$H^{\mathcal{C}^m} \leftarrow H^{\mathcal{C}^m} + \big[\,\mathrm{Attn}_{\mathrm{frame}}\!\big(\mathrm{LN}(H^{\mathcal{C}^m}_i);\, \theta\big)\,\big],\; i \in \{1, \dots, n\} \tag{Eq. 3}$$

(其中 $[\cdot]$ 為 concatenation operator,page 5)。

Implementation Details: 該模組權重來自 $\pi^3$ 預訓練 (page 6 Sec. 4.3 結尾)。paper does not specify head 數、隱藏維度與 dropout 設定。

Module 3: Sparse Sliding-Window Attention (SWA, Eq. 4)

Function: 提供跨 chunk 的 lossless 短程資訊高速公路,把上一個 chunk $\mathcal{C}^{m-1}$ 在同一 layer 的 token 直接餵給當前 chunk,避免單靠 TTT 壓縮所造成的失真 (page 4 倒數第二段「lossless short-range geometric alignment」)。

Input: 兩個相鄰 chunk 在同一 layer 的 token 串接 $[\mathrm{LN}(H^{\mathcal{C}^{m-1}}),\, \mathrm{LN}(H^{\mathcal{C}^m})]$,shape [B, M, 2nHW/p^2, d]。

Output: $H^{\mathcal{C}^m}$,shape [B, M, n*HW/p^2, d] (僅當前 chunk 被殘差更新)。

Processing: 在 $L$ 層 backbone 中只在 4 個 layer 插入 SWA (page 4「inserted at a subset of network depths (only four layers)」,page 5 (2)),query 取自 $\mathcal{C}^m$,key/value 同時涵蓋 $\mathcal{C}^{m-1} \cup \mathcal{C}^m$,因此邊界 frame 能直接複製鄰近高保真 features。SWA 不更新 $\mathcal{C}^{m-1}$ 的 tokens,維持單向 chunk-causal 結構。

Key Formulas:

$$H^{\mathcal{C}^m} \leftarrow H^{\mathcal{C}^m} + \mathrm{Attn}_{\mathrm{swa}}\!\big([\mathrm{LN}(H^{\mathcal{C}^{m-1}}),\, \mathrm{LN}(H^{\mathcal{C}^m})];\, \theta\big) \tag{Eq. 4}$$

Implementation Details: 第一個 chunk $\mathcal{C}^1$ 沒有前驅,SWA 在該 chunk 整層被略過 (Fig. 2 上半部 First Chunk 分支不含 SWA 連線)。paper does not specify 哪 4 個 layer index 被選為 SWA 插入點、window size、或 SWA 的 head 配置。

Module 4: Chunk-wise TTT Layer — Apply + Update (Eqs. 5-6)

Function: 在 chunk 軸上維護 $d \times d$ 的 fast-weight matrix $W^m$,作為長程、線性複雜度、有壓縮的全域記憶。一個 chunk 通過時先做 apply 把歷史記憶注入 token,再做 update 把當前 chunk 摘要寫回 $W^{m+1}$,完成「edit KV cache into $W$」式的 neural memory (page 3 Sec. 3「Test-Time Training」)。

Input:

  • 當前 chunk token $H^{\mathcal{C}^m}$,shape [B, M, n*HW/p^2, d];
  • 累積 fast weights $W^m$,shape [d, d]。

Output:

  • Apply 後 token $\bar{H}^{\mathcal{C}^m}$,shape 同輸入;
  • 下一 chunk 的 $W^{m+1}$,shape [d, d] (page 5 (3))。

Processing: 採用 LaCT (Large-Chunk TTT, Zhang et al., 2025b) 變體,以 chunk 為粒度執行 apply-then-update。Apply 將 LayerNorm 後的 $H^{\mathcal{C}^m}$ 餵入 fast-weight module $f_{W^m}$ 並殘差加回 (Eq. 5)。Update 把整個 chunk 視為一次 mini-batch,以 self-supervised key-value 重建為目標 (Sec. 3 Eq. 1 中 $\nabla_W \mathcal{L}(f_W(k), v)$),對 $W^m$ 做基於梯度的更新得到 $W^{m+1}$ (Eq. 6)。Pre-norm 套用於 TTT 內部以穩定 long-horizon streaming (page 5 (3)「We use pre-norm inside TTT」)。

Key Formulas:

$$\text{Apply:} \quad \bar{H}^{\mathcal{C}^m} = H^{\mathcal{C}^m} + f_{W^m}\!\big(\mathrm{LN}(H^{\mathcal{C}^m})\big) \tag{Eq. 5}$$

$$\text{Update:} \quad W^{m+1} = \mathcal{U}\!\big(W^m;\, H^{\mathcal{C}^m}\big) \tag{Eq. 6}$繼承 §3 的 TTT 通用 update rule:$W \leftarrow W - \eta\, \nabla_W \mathcal{L}\!\big(f_W(k),\, v\big) \tag{Eq. 1}$$

Implementation Details: $f_W$ 為 SwiGLU MLP,test-time update 採 Muon optimizer (Jordan et al., 2024) (page 4 Sec. 3 結尾)。第一個 chunk $\mathcal{C}^1$ 跳過 Apply,只執行 Update 以 bootstrap $W^2$ (Fig. 2 First Chunk 分支只有單一 Update box)。Inference 時可選擇每 5 個 window 重置 fast weights 以抑制 long-horizon drift (page 7 Sec. 5.1「we also reset the fast weights in the TTT layers after every five windows」)。paper does not specify 學習率 $\eta$、SwiGLU 隱藏維度、TTT loss 的具體解析形式以及 $W^1$ 的初始化方式。

Module 5: Chunk-wise Bidirectional Attention (Eq. 7)

Function: 在 chunk 內部恢復雙向 dense reasoning,讓 SWA / TTT 帶入的全域訊號與 chunk-內所有 frame 的 token 充分交互。

Input: TTT-apply 後的 token $\bar{H}^{\mathcal{C}^m}$,shape [B, M, n*HW/p^2, d]。

Output: 進入下一個 block 的 $H^{\mathcal{C}^m}$,shape 同輸入。

Processing: 對整個 chunk 的 token 串 (跨 frame) 執行 bidirectional self-attention,LayerNorm + 殘差 (page 5 (4)「powerful geometric reasoning under a bounded context window」)。bounded context window = $n \cdot HW/p^2$ tokens,確保計算成本不隨 $T$ 成長。

Key Formulas:

$$H^{\mathcal{C}^m} \leftarrow \bar{H}^{\mathcal{C}^m} + \mathrm{BiAttn}_{\mathrm{chunk}}\!\big(\mathrm{LN}(\bar{H}^{\mathcal{C}^m});\, \theta\big) \tag{Eq. 7}$$

Implementation Details: 該模組權重亦自 $\pi^3$ 初始化 (page 6 Sec. 4.3 結尾「we initialize the weights of the patchifier, frame attention, and chunk-wise bidirectional attention modules from $\pi^3$」)。paper does not specify head 數與是否使用 RoPE 等 positional encoding。

Module 6: Prediction Heads (Pointmap + Camera-Pose Decoders)

Function: 在最後一個 residual block 之後,從 token 序列解碼出每 frame 的 dense local pointmap 與 camera pose,沿用 $\pi^3$ 的設計。

Input: 最後一層 token 張量 $H^{\mathcal{C}^m,(L)}_j$,shape [B, M, n, HW/p^2, d]。

Output:

  • Per-frame local pointmap $P_i \in \mathbb{R}^{H \times W \times 3}$,聚合為 [B, T, H, W, 3];
  • Per-frame camera pose $c_i \in \mathbb{R}^{4 \times 4}$,聚合為 [B, T, 4, 4] (page 3 Sec. 3 開頭定義)。

Processing: 「lightweight decoders」並列掛在 backbone 之後,pointmap decoder 與 camera-pose decoder 各自從共享 token 表徵預測對應 modality (page 5「After the final residual block, we attach lightweight decoders ... following $\pi^3$」)。

Key Formulas: paper does not specify decoder 內部公式; 預測的 local point map 透過 $\Pi(\mathbf{T}, \mathbf{x})$ 投影到 world coordinates 用於 $\mathcal{L}_{\mathrm{global}}$ (Eq. 10),預測的相對 pose $\hat{R}_{ij}, \hat{t}_{ij}$ 則被 Eq. 9 同時以 rotation loss 與 Huber translation loss 監督。

Implementation Details: 整個 decoder 部分未從 $\pi^3$ 初始化 — Sec. 4.3 結尾僅列出 patchifier、frame attention、chunk-wise bidirectional attention 三者使用 $\pi^3$ 權重 (page 6)。paper does not specify decoder 的層數、隱藏維度與 head 拆分方式。

Module 7: LoGeR* Feedforward SE(3) Alignment (Eq. 12, optional)

Function: 在 inference 與訓練時對相鄰 chunk 之間做剛體對齊,把 raw 預測 pose 旋進一致的全域座標系,進一步抑制超長 horizon 上的累積漂移。

Input:

  • 當前 chunk 的 raw 預測 pose $\hat{T}^{(m)}_t$,$t \in \mathcal{C}^m$,shape [B, n, 4, 4];
  • 上一 chunk 已對齊的 pose $\tilde{T}^{(m-1)}_k$,$k$ 為 overlapping frame index。

Output: 對齊後 pose $\tilde{T}^{(m)}_t$,shape 同輸入,並設定 $\tilde{T}^{(1)} = \hat{T}^{(1)}$ 作為遞推起點 (page 5 結尾)。

Processing: 以 overlapping frame $k$ 上的 pose 計算 SE(3) 剛體變換 $A_m$,再左乘到整個 chunk $\mathcal{C}^m$ 的所有 frame pose 上。整體為純前饋運算,不含迭代最佳化或 bundle adjustment。

Key Formulas:

$$A_m = \tilde{T}^{(m-1)}_k \big(\hat{T}^{(m)}_k\big)^{-1}, \quad A_m \in \mathrm{SE}(3)$$

$$\tilde{T}^{(m)}_t = A_m \cdot \hat{T}^{(m)}_t, \quad \forall\, t \in \mathcal{C}^m \tag{Eq. 12}$$

Implementation Details: LoGeR* 從第一階段的 LoGeR checkpoint 初始化,將 alignment 模組整合進剩餘 curriculum 並 fine-tune (page 6 Sec. 4.3 結尾)。Inference 期間若搭配 fast-weights periodic reset 也會同時觸發此 alignment (page 7 Sec. 5.1)。paper does not specify overlap frame 數量 (主文僅提「single overlapping frame」,即 $k$ 為 chunk 邊界唯一共享 frame)。

6.3 Training and Inference Settings

  • Hardware: 前期 32 張 NVIDIA H100 約兩天,後接 32 張 NVIDIA H200 約兩天;所有評估在單張 NVIDIA A100 (40GB) 上進行。
  • Batch size: 32。
  • Optimizer: AdamW (Loshchilov & Hutter, 2019),$\beta = (0.9, 0.999)$。
  • Learning rate: decoupled — 新引入的 TTT 與 SWA 層為 $5 \times 10^{-4}$,unfrozen 的 base network 參數為 $1 \times 10^{-5}$;cosine decay 搭配 1,000 steps warmup (Appendix A.3)。
  • Weight decay: 0.05,僅施加於 $\geq 2$ 維的參數張量(排除 bias 與 LayerNorm scale/shift)。
  • Loss weights: $\lambda_r = 0.1$、$\lambda_t = 10$、$\lambda_{\text{global}} = 1$。
  • Training steps: 共 40k steps;Stage 1 在 H100 訓練 25,000 steps,固定 48 frames,chunk size 從 12 線性降至 4、overlap 從 3 降至 1;Stage 2 在 H200 訓練 15,000 steps,將序列長度延伸至 128 frames,chunk size 從 12 降至 8、overlap 從 3 降至 2。
  • Initialization: patchifier、frame attention、chunk-wise bidirectional attention 從 $\pi^3$ (Wang et al., 2026) 初始化;encoder 與 prediction heads 訓練時凍結。
  • Architecture spec (Appendix A.2): 18 個 residual blocks,每個 block 都插入 TTT layer;SWA layer 僅插在第 6、10、14、18 個 block;TTT head dim = 512、expansion factor = 4;新增三個可學習的 positional embedding tokens 用以標示與前/後 chunk 的 overlap 狀態;base network 約 954M 參數,新增 TTT + SWA 層約 296M 參數。
  • Efficiency tricks: gradient checkpointing,SWA 以 FlexAttention (Dong et al., 2024) 實作;推論時對 SWA 啟用 KV-cache。
  • Inference settings (Appendix A.4): 短序列 LoGeR 與 LoGeR 皆用 chunk size 64、overlap 3;長序列上 LoGeR 維持此設定,LoGeR 在 KITTI 用 chunk size 32、在 VBR 用 chunk size 48;TTT fast weights 在每 5 個 window 後 reset 一次以避免錯誤累積,reset 時亦套用 feedforward pose alignment。

6.5 Ablation Studies

所有 ablation 在縮減 frame 數的設定下訓練,評估在 ScanNet 子集與 TUM-Dynamics 上,報告 500/1000 frames 的 ATE (Tab. 3)。

  • w/o TTT (移除全域 fast-weight memory):ScanNet 1000f 從 0.107 升至 0.162、TUM 1000f 從 0.050 升至 0.079;Fig. 10 顯示去掉 TTT 後在長視野出現嚴重 trajectory drift。這是 diagnostic 級別的測試,直接驗證「TTT 負責 global anchoring」的論點。
  • w/o SWA (移除短期 lossless 對齊):ScanNet 1000f 升至 0.143、TUM 1000f 升至 0.053;Fig. 10 顯示鄰接 chunk 的局部對齊出現可見錯位。同樣對應到「SWA 負責 local 高保真對齊」的設計動機,診斷性充分。
  • w/o 5 large datasets (移除 TartanAir、TartanAirV2、Waymo、Virtual KITTI 2、OmniWorld-Game):ScanNet 1000f 從 0.107 升至 0.156、TUM 1000f 從 0.050 升至 0.072,直接支持 §4.3 的「data wall」論點。
  • w/o curriculum (LoGeR):ScanNet 1000f 從 0.107 升至 0.133、TUM 1000f 從 0.050 升至 0.062,證明漸進式 curriculum 對 recurrent TTT 收斂的必要性。
  • **w/o curriculum (LoGeR\*)**:ScanNet 1000f 從 0.080 升至 0.093,但 TUM 500f 反而 0.029 < 0.031;表示 curriculum 對 LoGeR* 的增益主要來自 ScanNet,在 TUM 短序列上效益不明顯。

整體來看,SWA / TTT 兩項屬於 component-level diagnostic ablation(配合 Fig. 10 的視覺證據可解釋失敗模式),而 dataset 與 curriculum 兩項則屬於 design-choice ablation;並未出現純粹「沒做這件事就更差」的 sanity check。缺口:沒有對 chunk size、overlap size、SWA 插入深度數量(只在 4 個 block)、TTT head dimension/expansion factor 等架構超參做敏感度分析,也沒有 ablate fast-weight 的 update rule 選擇 (Muon vs. 其他)。

7. Phyra's Judgment

7.1 Claimed vs. Supported Contributions

  • C1.「Hybrid memory（SWA + TTT）能同時維持 lossless 局部對齊與 compressed 全域一致性」 Partially supported。Tab. 3 的 ablation（p. 9）與 Fig. 10（p. 9）的 trajectory 視覺化確實顯示拿掉 SWA 會造成局部 misalignment、拿掉 TTT 會造成 trajectory drift，符合定性敘述；但在最終長序列 inference 時，全域 anchoring 大量依賴 LoGeR$^{*}$ 的 SE(3) alignment 與 $5$-chunk reset（Tab. 2、Tab. 6），TTT 單獨的 anchoring 強度未被獨立量化。
  • C2.「Trained on 128 frames, generalize to 19k frames」 Partially supported。Fig. 4（p. 7）與 Tab. 6（p. 15）確實顯示模型可以一路推到 $\sim 19$k frame 並仍是 SOTA，但 inference 時要靠 periodic state reset 與 feedforward alignment（Sec. 5.1，p. 7；Appendix A.4，p. 14），所以「generalization」其實是「在重置策略下保持可用」，而非 TTT 自身的 length generalization。
  • C3.「KITTI ATE 比前一代 feedforward 降低逾 $74\%$」 Supported。Tab. 2（p. 7）的 $72.86 \to 18.65$ 計算正確，且 baseline 設定（TTT3R、CUT3R、FastVGGT 等）涵蓋當前主要 feedforward family；但平均值受 KITTI 01（無 loop、$2.5$ km）這類 baseline 嚴重失敗的 case 拉動，per-sequence 上對比 optimization-based 的 DPV-SLAM++ 在含 loop 序列上仍輸。
  • C4.「Curriculum + 大尺度資料是必要條件」 Supported。Tab. 3（p. 9）的 dataset ablation 與 curriculum ablation 在兩個 dataset、兩個序列長度下一致顯示退化，且 Fig. 3（p. 5）的 FastVGGT 在 VBR 上完全失敗的對照，把 data wall 的論點以視覺化形式展示。
  • C5.「Pi3-Chunk 是一個強 baseline，hybrid memory 在其上 stacking 仍有顯著 gain」 Supported on long sequences, weakly supported on short ones。Fig. 4（p. 7）與 Tab. 6（p. 15）顯示在 $\geq 4$k frame 區段 LoGeR 顯著勝過 Pi3-Chunk；但在短序列（TUM-Dynamics、$\sim 200$–$500$ frame）的 Fig. 9（p. 8）兩者差距很小，C5 在中短序列上是 weak 的。

7.2 Fundamental Limitations of the Method

沒有任何 loop closure 機制。 Chunk-causal pipeline 加上 TTT 是純粹的 forward 累積結構，當相機回到舊位置時，沒有任何模組會觸發跨大時間距離的特徵 re-association。即使 fast weight 已經把先前訪問編碼進記憶，它的更新規則（梯度式、單向）也只能把新 chunk 的資訊寫入，不會在「偵測到 revisit」時把當前 pose 拉回舊座標。這在 KITTI 含 loop 序列（00、05、06）上以數量級的差距表現出來：DPV-SLAM++ $8.30$/$5.74$/$11.60$ 對 LoGeR$^{*}$ $30.47$/$26.34$/$6.60$（Tab. 2，p. 7）。在此架構下，再多訓練資料也無法補上這個缺口。

TTT memory 的「無限 context」是名義的、不是實效的。 因為 fast weight 的訓練 chunk 數有限（curriculum 最高到 $20$ chunks），inference 超過後 length generalization 直接失敗（Discussion，p. 9 引用 Ruiz & Gu, 2025），必須每 $5$ window reset。Reset 等於丟棄全部累積的全域記憶，所以對 $19$k frame 序列而言，模型實際擁有的「全域 anchoring」只是當下一段約 $5$ chunk 的 TTT 狀態加上 deterministic SE(3) alignment 的鏈式傳遞，與一個強化版的 sliding stitcher 結構本質類似。這個限制不是 LoGeR 獨有的工程取捨，而是現階段所有 fast-weight memory 的共同瓶頸。

全域座標的 anchoring 實質上由 feedforward alignment 完成。 LoGeR$^{}$ 在幾乎所有 benchmark 都優於 base LoGeR（Tab. 2、Tab. 6、Tab. 3），而 LoGeR$^{}$ 與 base 的唯一差異是把 chunk 間 overlap frame 用 SE(3) 解析對齊。換言之，論文宣稱的「parametric long-range memory anchors the global coordinate frame」在實作上需要一個 closed-form 對齊作為兜底。這意味著 hybrid memory 的真正功能更像「讓 chunk 間的 dense feature 不要漂移得太離譜，方便對齊」，而不是「自身就完成全域對齊」，這與架構敘事有微妙落差。

Dense vision token 密度使 hybrid 方案無法照搬 LM 經驗。 論文自己在相關工作中指出 Jamba/Longformer 之所以可以混用 full attention + 線性方案，是因為 LM token 數可控；dense vision 一張 $504\times 280$ 圖就有大量 patch，所以 LoGeR 完全放棄了 long-range 的 full attention（連 sparse global 都沒有）。這帶來一個結構性 ceiling：任何需要「跨整個影片做 dense token 比對」的 task（例如真正的 dense loop closure、跨大時間 view synthesis）都無法在此架構下表達。

7.3 Citations Worth Tracking

  • π3 (Wang et al., 2026)：LoGeR 的 backbone、loss、decoder 全部來自 π3，理解 LoGeR 等於 「π3 + chunk + hybrid memory」，所以必須先掌握 π3 的 permutation-equivariant 雙向幾何學設計，才能判斷 LoGeR 的增益是來自 memory 還是 backbone。
  • **LaCT (Zhang et al., 2025b, Test-Time Training Done Right)**：本文的 chunk-wise TTT 是 LaCT 的直接調用，所有對 fast-weight 的 update/apply 行為、Muon optimizer 選擇、SwiGLU 設計都奠基於此；想要評估 TTT memory 容量、reset 必要性的根因，必讀此文。
  • TTT3R (Chen et al., 2026)：concurrent 的 frame-wise TTT 3D 重建，是 LoGeR 最直接的對照組，理解兩者「frame-wise 線性 vs chunk-wise hybrid」的差異，可以判斷 hybrid memory 額外好處的來源是「chunk-wise」還是「+SWA」。
  • VBR dataset (Brizi et al., 2024)：本文新建 long-context benchmark 的資料來源，包含 LiDAR ground truth 與 bundle-adjusted poses 的細節決定了評測的公平性與可重現性，要重做此實驗必先理解 VBR 原始協議。
  • **Ruiz & Gu, 2025, *Understanding and improving length generalization in recurrent models***：作者明確引用此文作為「TTT 在超出訓練 chunk 數後 fail」與「需要 periodic reset」的依據（Discussion，p. 9），是判斷 LoGeR 真實 long-horizon 上限的理論參照。

8. Open Questions and Improvement Ideas

8.1 Outstanding Questions

  • [ ] 若拿掉 LoGeR$^{*}$ 的 SE(3) feedforward alignment，但保留 hybrid memory，KITTI 與 VBR 的 ATE 退化幅度是多少？這直接決定 TTT 全域 anchoring 的實效強度。
  • [ ] Periodic state reset 的頻率（目前固定 $5$ windows）對最終 ATE 的影響曲線長什麼樣？是否存在一個 reset 間隔讓 base LoGeR 在不需要 LoGeR$^{*}$ alignment 的情況下達到相同效能？
  • [ ] 把 LoGeR 的 backbone+memory 做 parameter-matched 對照（例如把 π3 直接放大到 $\sim 1.25$B 後再做 chunk-stitch），hybrid memory 的純架構增益還有多少？
  • [ ] VBR 之外、跨城市或非結構化戶外（如自然森林、海岸）長序列上，LoGeR 是否仍能保持 $19$k frame 的穩定？目前所有 long-horizon evidence 都來自 Rome 同分布序列。
  • [ ] TTT fast weight 是否能透過 retrieval-style 機制重新讀取 reset 前的歷史（例如把舊 fast weight 序列化為 KV cache），以解除 $5$-chunk 的硬截斷？
  • [ ] 在含 loop 的 KITTI 序列上，把 SWA 的 window 從「相鄰 chunk」擴張到「位置最相近的舊 chunk」（feature-similarity 索引），是否能在不引入 backend optimization 的前提下取得隱式 loop closure？
  • [ ] 推論時 chunk size 對長序列敏感（KITTI 須降到 $32$、VBR 須降到 $48$），這個敏感性的根因是 TTT 的容量、SWA 的記憶連續性、還是訓練 chunk 分布？目前論文未拆解。

8.2 Improvement Directions

  1. Feature-similarity-driven SWA 取代純鄰接 SWA。 目前 SWA 只看 $C^{m-1} \cup C^{m}$，這是時間鄰接而非空間鄰接。若在 SWA 內把候選 key 池擴張為「最近的 $k$ 個 + 與當前 query 在 DINO embedding 上最相似的 $k'$ 個 chunk」，等於在 feedforward pipeline 內塞入弱化版 loop closure，且仍維持 $O(N)$。這是 KITTI 含 loop 序列上輸給 DPV-SLAM++ 的最直接補救路徑（基於 7.2 第一段的結構性缺口）。
  2. Multi-rate TTT（hierarchical fast weights）。 並行維護兩組 fast weight：一組 reset 間隔 $5$ window 維持高精度局部全域、一組 reset 間隔 $20$+ window 用更激進的低 learning rate 與 weight decay 維持 coarse global scale。這直接呼應 7.2 第二段提到的 length generalization 瓶頸：用 timescale 分離取代單一 fast weight，理論上能讓 $19$k frame 不再依賴 reset 與 SE(3) alignment。
  3. **把 LoGeR$^{*}$ 的 SE(3) alignment 改為 learned residual。** 既然實驗顯示 LoGeR$^{*}$ 幾乎一律優於 base LoGeR，與其在 inference 時做 closed-form alignment，不如把它做成一個 chunk-level residual transformation head，讓 TTT 的 update rule 直接學習 anchoring 殘差。這能把目前外掛、解析的對齊整合進端到端訓練，緩解 7.1-C1 的「hybrid memory 並未真正 anchor」問題。
  4. Parameter-matched 與 Pi3-Chunk-with-extra-params 的 ablation。 現有比較未控制總參數量。最便宜的補強是訓練一個和 LoGeR 等大（$\sim 1.25$B）的 Pi3-Chunk 變體，把參數放到 BiAttn/decoder 上，驗證 hybrid memory 的增益不是來自「多了 $296$M 參數」。這不需要新架構，純粹補實驗，但對外界可信度提升大。
  5. Cross-domain long-horizon benchmark（非 Rome）。 VBR 的 generalization 範圍未驗證（4.2.2 第三點）。建議再加上至少一個 non-Rome long sequence（例如 KITTI-360 全鎮、Waymo 的最長 city-scale 軌跡、或 OmniWorld-Game 的長序列）作為 holdout，以確認「19k frame」不是只在訓練分布外推上有效。這純粹是評測擴增、無需改動架構，feasibility 最高。
